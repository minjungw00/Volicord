# 이후 후보: 보안과 보증

## 이 문서가 담당하는 것

이 문서는 더 강한 보안 자세, 보증 주장, 역량 라벨, 관찰, 차단, 검증 신뢰도에 관한 비활성 이후 후보를 담당합니다. 후보를 이곳에 묶어 두면 [이후 후보 색인](index.md)은 경로 안내와 짧은 요약으로 남을 수 있습니다.

여기의 모든 후보는 미래 방향의 문서 원천 자료일 뿐입니다. 런타임 동작을 활성화하지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP 보안 보장, 현재 접근 등급, 현재 API 메서드, 저장 효과, UI 동작, 커넥터 동작, 실행 가능한 적합성, 구현 준비 상태를 정의하지 않습니다.

또한 더 강한 보장이 가능하다고 결정하지도 않습니다. 예방형, 탐지형, 격리, 가림, 관찰, 차단 주장은 승격 시점에 현재 보안 담당 문서와 현재 범위 담당 문서가 다시 맡아야 합니다.

## 카테고리 경계

이 카테고리의 중심 질문은 "하네스가 어떤 보증을 정직하게 주장할 수 있는가?"입니다. 예방형 통제 후보, 격리 라벨, 역량 프로필 강화, 명령/네트워크/비밀값 관찰, 도구 실행 전 차단, 더 강한 검증 신뢰도 주장을 포함합니다.

아티팩트 저장 메커니즘으로서의 네이티브 아티팩트 캡처, 커넥터 접점 설계, 팀 작업 흐름, 검증기 카탈로그 세부사항은 이 문서가 담당하지 않습니다. 단, 후보의 핵심이 보증 주장인 경우에는 승격 전 보안과 보증 관점만 이 문서가 담당합니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 보증 강화 | 현재 MVP보다 강한 증거, 검증, 닫기 준비 상태 보증 주장. |
| 운영 강화 | 로컬 운영 진단과 더 강한 보안 자세. |
| 더 강한 로컬 역량 프로필 | 관찰, 캡처, 격리, 차단 역량을 나타내는 향후 프로필 라벨. |
| 명령, 네트워크, 비밀값 접근 관찰 | 선택된 명령, 네트워크, 비밀값 접근 의도 관찰. |
| 명령, 네트워크, 비밀값 도구 실행 전 차단 | 도구 실행 전 예방형 차단 주장. |
| 역량 조건부 `prepare_write`와 `record_run` 관찰 | 쓰기 준비와 실행 기록 주변의 명령, 네트워크, 비밀값 접근 관찰. |
| 역량 프로필 지원 필드 | 관찰, 캡처, 도구 실행 전 차단, 격리 역량 지원 필드. |
| 역량 조건부 권한 관찰 필드 | `intended_commands`, `intended_network`, `network_write`, `secret_access` 같은 향후 필드. |
| 이후 닫기와 보증 필드 | 닫기, 관문, 검증, QA, 설계, 보증 필드. |
| 네이티브 후크와 고급 사이드카 감시 | 더 넓은 도구 가시성을 위한 네이티브 후크나 사이드카 감시 주장. |

## 후보 상세

<a id="assurance-hardening"></a>
### 보증 강화

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 관문, 검증기, 닫기 준비 상태 요구사항을 만들지 않습니다.
- 승격 시 초점: 더 강한 보증이 규범이 된다면 보증 담당 문서, 스키마 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

<a id="operations-hardening"></a>
### 운영 강화

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 운영자 명령, 진단, 보안 보장을 만들지 않습니다.
- 승격 시 초점: 로컬 운영 강화가 규범이 된다면 운영 담당 문서, 보안 표현, API 동작, 적합성 점검을 함께 다룹니다.

<a id="stronger-local-capability-profiles"></a>
### 더 강한 로컬 역량 프로필

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 예방형, 격리형, 관찰, 캡처, 도구 실행 전 차단, 격리 보장을 만들지 않습니다.
- 승격 시 초점: 역량 프로필이 승격된다면 에이전트 통합 담당 문서, 보안 담당 문서, 스키마 담당 문서, 적합성 점검을 함께 다룹니다.

<a id="command-network-and-secret-access-observation"></a>
### 명령, 네트워크, 비밀값 접근 관찰

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 명령 관찰, 네트워크 관찰, 비밀값 접근 관찰 권한을 만들지 않습니다.
- 승격 시 초점: 관찰 주장이 승격된다면 에이전트 통합 담당 문서, 보안 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

<a id="command-network-and-secret-pre-tool-blocking"></a>
### 명령, 네트워크, 비밀값 도구 실행 전 차단

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 예방형 차단, 격리, 임의 도구 제어 보장을 만들지 않습니다.
- 승격 시 초점: 차단 주장이 승격된다면 보안 담당 문서, 에이전트 통합 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

<a id="capability-gated-prepare-write-and-record-run-observation"></a>
### 역량 조건부 `prepare_write`와 `record_run` 관찰

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `prepare_write`나 `record_run`에 현재 명령, 네트워크, 비밀값 접근 관찰을 만들지 않습니다.
- 승격 시 초점: 관찰 분기가 승격된다면 API 동작, 보안 담당 문서, 스키마 담당 문서, 적합성 점검을 함께 다룹니다.

<a id="capability-profile-support-fields"></a>
### 역량 프로필 지원 필드

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `command_observation_supported`, `network_observation_supported`, `secret_access_observation_supported`, `artifact_capture_supported`, `pre_tool_blocking_supported`, `isolation_supported`를 활성화하지 않습니다.
- 승격 시 초점: 지원 필드가 승격된다면 스키마 담당 문서, 에이전트 통합 담당 문서, 보안 담당 문서, 적합성 점검을 함께 다룹니다.

<a id="capability-gated-authorization-observation-fields"></a>
### 역량 조건부 권한 관찰 필드

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `intended_commands`, `intended_network`, `intended_secret_scope`, `network_write`, `external_service_write`, `secret_access`를 활성화하지 않습니다.
- 승격 시 초점: 관찰 필드가 승격된다면 스키마 담당 문서, API 동작, 보안 담당 문서, 적합성 점검을 함께 다룹니다.

<a id="later-close-and-assurance-fields"></a>
### 이후 닫기와 보증 필드

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `verifying`, `qa`, `completed_verified`, `detached_verified`, `design_gate`, `verification_gate`, `qa_gate`, 수동 QA 관문 필드, 설계 정책 차단 사유, 보증 차단 사유를 활성화하지 않습니다.
- 승격 시 초점: 닫기나 보증 필드가 승격된다면 스키마 담당 문서, Core 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

<a id="native-hooks-and-advanced-sidecar-watcher"></a>
### 네이티브 후크와 고급 사이드카 감시

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 예방형 가드 확장, 네이티브 후크, 사이드카 감시, 임의 도구 제어 보장을 만들지 않습니다.
- 승격 시 초점: 후크나 감시 주장이 승격된다면 보안 담당 문서, 에이전트 통합 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

## 승격 규칙

이 파일만 고쳐서는 후보가 승격되지 않습니다. 후보는 현재 범위와 관련 현재 담당 문서가 같은 문서 전용 작업 묶음에서 갱신될 때만 활성화될 수 있습니다.

승격할 동작에 현재 담당 문서가 없다면, 그 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기에 후보가 언급되어도 보장, 프로필, 필드, 메서드, 접근 등급, 검증기, fixture, UI, 런타임 통제가 활성화되지 않습니다.
