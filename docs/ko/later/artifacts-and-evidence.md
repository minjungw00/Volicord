# 이후 후보: 아티팩트와 증거

## 이 문서가 담당하는 것

이 문서는 더 풍부한 증거, 네이티브 아티팩트 캡처, 아티팩트 본문 정책, 내보낼 수 있는 증거 패키지, 브라우저 캡처, 증거 관련 표시나 템플릿 확장에 관한 비활성 이후 후보를 담당합니다. 후보를 이곳에 묶어 두면 [이후 후보 색인](index.md)은 경로 안내와 짧은 요약으로 남을 수 있습니다.

여기의 모든 후보는 미래 방향의 문서 원천 자료일 뿐입니다. 현재 증거, 아티팩트, 저장소, 닫기 준비 상태 동작을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP 아티팩트 저장 동작, 현재 아티팩트 본문 읽기, 증거 충분성, 현재 API 메서드, 저장 효과, UI 동작, 보안 보장, 적합성 요구사항을 정의하지 않습니다.

또한 아티팩트 캡처를 증거 수락과 같은 것으로 만들지 않습니다. 승격 시에는 Core 소유 상태, 아티팩트, 사용자 소유 판단, 검증, 수락, 잔여 위험의 경계를 유지해야 합니다.

## 카테고리 경계

이 카테고리의 중심 질문은 "하네스가 나중에 어떤 더 풍부한 자료를 기록, 참조, 패키징, 요약할 수 있는가?"입니다. 네이티브 캡처, 전체 `Evidence Manifest`, 내보내기 묶음, 아티팩트 본문 정책, 브라우저 캡처 증거, 증거 관련 템플릿 이름을 포함합니다.

보안 통제 주장, 커넥터 세부 동작, 검증기 카탈로그, 협업 생명주기는 이 문서가 담당하지 않습니다. 단, 승격 전 아티팩트나 증거 자료를 설명하는 데 필요한 범위에서는 짧게 언급할 수 있습니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 전체 `Evidence Manifest` | 매니페스트 수준 증거 기록과 렌더링 요약. |
| 내보내기 | 내보내기 동작, 내보내기 아티팩트, 가림 경계. |
| 내보내기와 인계 형식 | 내보내기나 인계를 위한 파일 형식, 번들 계약, 출처 추적 요구사항. |
| 이후 행위자, 생산자, 캡처 출처 값 | `evaluator`, `operator`, `capture_adapter` 같은 생산자, 행위자, 캡처 출처 값. |
| 네이티브 아티팩트 캡처 | 참조 전용 아티팩트 기록을 넘어 아티팩트 본문을 캡처하는 후보. |
| 읽기 전용 플레이북과 판단 맥락 메타데이터 | 판단을 그 자체로 충족하지 않으면서 증거 검토를 도울 수 있는 읽기 전용 메타데이터. |
| 이후 참조와 아티팩트 값 계열 | 번들, 매니페스트, QA 캡처, 내보내기 구성요소, 설계, `Eval`, 수동 QA, TDD, `Projection`, 관련 참조 값 계열. |
| 이후 템플릿 이름 | 더 풍부한 판단, 증거, 실행, 설계, 내보내기 표시를 위한 템플릿 이름. |
| 브라우저 캡처 자동화 | 브라우저 스크린샷, 녹화, 캡처된 UI 상태를 증거 자료로 다루는 후보. |

## 후보 상세

<a id="full-evidence-manifest"></a>
### 전체 `Evidence Manifest`

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 증거, 아티팩트, 닫기 준비 상태 요구사항을 만들지 않습니다.
- 승격 시 초점: 매니페스트 수준 증거가 규범이 된다면 증거 담당 문서, 스키마 담당 문서, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="export"></a>
### 내보내기

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 내보내기 동작, 내보내기 아티팩트, 가림 보장을 만들지 않습니다.
- 승격 시 초점: 내보내기가 규범이 된다면 내보내기 담당 문서, 아티팩트 담당 문서, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="export-and-handoff-formats"></a>
### 내보내기와 인계 형식

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 파일 형식, 번들 계약, 출처 추적 요구사항을 만들지 않습니다.
- 승격 시 초점: 형식이 승격된다면 내보내기 담당 문서, 인계 담당 문서, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="later-actor-producer-and-capture-source-values"></a>
### 이후 행위자, 생산자, 캡처 출처 값

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `evaluator`, `operator`, `capture_adapter`를 활성화하지 않습니다.
- 승격 시 초점: 행위자, 생산자, 캡처 출처 값이 승격된다면 스키마 담당 문서, API 동작, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="native-artifact-capture"></a>
### 네이티브 아티팩트 캡처

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 API 요구사항을 만들지 않습니다.
- 승격 시 초점: 하네스가 참조 스테이징만이 아니라 아티팩트 데이터를 직접 캡처하게 된다면 스키마 담당 문서, API 동작, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="read-only-playbook-and-judgment-context-metadata"></a>
### 읽기 전용 플레이북과 판단 맥락 메타데이터

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 읽기 전용 메타데이터 요구사항을 만들지 않으며 그 자체로 어떤 판단도 충족하지 않습니다.
- 승격 시 초점: 읽기 전용 맥락 메타데이터가 승격된다면 에이전트 통합 담당 문서, API 동작, 스키마 담당 문서, 적합성 점검을 함께 다룹니다.

<a id="later-reference-and-artifact-value-families"></a>
### 이후 참조와 아티팩트 값 계열

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 이후 번들, 매니페스트, QA 캡처, 내보내기 구성요소, 설계, `Eval`, 수동 QA, TDD, 상태 보기, 관련 참조 값 계열을 활성화하지 않습니다.
- 승격 시 초점: 값 계열이 승격된다면 아티팩트 담당 문서, 스키마 담당 문서, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="later-template-names"></a>
### 이후 템플릿 이름

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `DEC`, `APR`, `RUN-SUMMARY`, `EVIDENCE-MANIFEST`, `EVAL`, `MANUAL-QA`, `TASK`, `DIRECT-RESULT`, `JOURNEY-CARD`, `DESIGN`, `DOMAIN-LANGUAGE`, `MODULE-MAP`, `INTERFACE-CONTRACT`, `TDD-TRACE`, `EXPORT` 템플릿을 활성화하지 않습니다.
- 승격 시 초점: 템플릿 이름이나 표시 본문이 승격된다면 템플릿 담당 문서, 원천 기록 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

<a id="browser-capture-automation"></a>
### 브라우저 캡처 자동화

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 브라우저 캡처, 보존, 가림, QA, 수락 동작을 만들지 않습니다.
- 승격 시 초점: 브라우저 캡처가 규범적 증거 자료가 된다면 캡처 담당 문서, 보안 담당 문서, 저장 효과, 적합성 점검을 함께 다룹니다.

## 승격 규칙

이 파일만 고쳐서는 후보가 승격되지 않습니다. 후보는 현재 범위와 관련 현재 담당 문서가 같은 문서 전용 작업 묶음에서 갱신될 때만 활성화될 수 있습니다.

승격할 동작에 현재 담당 문서가 없다면, 그 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기에 후보가 언급되어도 아티팩트 캡처, 증거 수락, 본문 저장, 내보내기 형식, 템플릿 본문, UI 카드, 적합성 기대가 활성화되지 않습니다.
