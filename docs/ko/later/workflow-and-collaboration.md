# 이후 후보: 작업 흐름과 협업

## 이 문서가 담당하는 것

이 문서는 더 풍부한 작업 구체화, 다음 행동 흐름, 검증과 평가 작업 흐름, 사용자 판단 분기, 위험 검토, 릴리스 인계, `reconcile`, 팀 오케스트레이션, 협업 생명주기에 관한 비활성 이후 후보를 담당합니다. 후보를 이곳에 묶어 두면 [이후 후보 색인](index.md)은 경로 안내와 짧은 요약으로 남을 수 있습니다.

여기의 모든 후보는 미래 방향의 문서 원천 자료일 뿐입니다. 현재 작업 흐름, API, 저장소, UI, 협업 요구사항을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP API 동작, 작업 생명주기, 닫기 준비 상태 동작, 수락 동작, 잔여 위험 수락, 팀 권한, 릴리스 자동화, 저장 효과, 보안 보장, 적합성 동작, 구현 준비 상태를 정의하지 않습니다.

또한 이후 판단 분기를 사용자 소유 판단의 대체물로 만들지 않습니다. 민감 동작 승인, 최종 수락, 잔여 위험 수락, 면제, 조정, 검증 위험 수락은 관련 현재 담당 문서가 승격 작업에서 명시적으로 바꾸지 않는 한 서로 구분됩니다.

## 카테고리 경계

이 카테고리의 중심 질문은 "하네스가 나중에 사람, 에이전트, 판단, 생명주기 단계를 어떻게 조율할 수 있는가?"입니다. 구체화 기록, 다음 행동, 더 풍부한 검증과 평가 흐름, `Decision Packet` 표시, 위험 검토, `reconcile`, 릴리스 인계, 팀 작업 흐름, 파생 작업 맥락을 포함합니다.

커넥터 세부 동작, 아티팩트 본문 저장, 보안 보장, 검증기 카탈로그는 이 문서가 담당하지 않습니다. 향후 작업 흐름이 그 영역에 의존하더라도 이 문서는 승격 전 작업 흐름과 협업 관점의 후보만 기록합니다.

### 고급 릴리스와 배포 자동화

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 릴리스, 배포, 카나리, 롤백, 병합, 운영 환경 모니터링 자동화를 만들지 않습니다.
- 승격 시 초점: 배포 자동화가 승격된다면 릴리스 담당 문서, 보안 담당 문서, API 동작, 적합성 점검을 함께 다룹니다.

## 승격 규칙

이 파일만 고쳐서는 후보가 승격되지 않습니다. 후보는 현재 범위와 관련 현재 담당 문서가 같은 문서 전용 작업 묶음에서 갱신될 때만 활성화될 수 있습니다.

승격할 동작에 현재 담당 문서가 없다면, 그 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기에 후보가 언급되어도 작업 흐름 분기, API 메서드, 상태 전이, 판단 경로, 권한 모델, 릴리스 자동화, 팀 동작, UI, 저장 효과, 닫기 준비 상태 요구사항이 활성화되지 않습니다.
