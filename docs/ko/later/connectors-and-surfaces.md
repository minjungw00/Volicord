# 이후 후보: 커넥터와 접점

## 이 문서가 담당하는 것

이 문서는 향후 IDE, CLI, 채팅, MCP, 호스팅, 원격, 대시보드, 커넥터 대상 접점에 관한 비활성 이후 후보를 담당합니다. 후보를 이곳에 묶어 두면 [이후 후보 색인](index.md)은 경로 안내와 짧은 요약으로 남을 수 있습니다.

여기의 모든 후보는 미래 방향의 문서 원천 자료일 뿐입니다. 현재 접점 지원, 커넥터 권한, 호스팅 런타임 동작, UI 요구사항을 만들지 않습니다.

## 이 문서가 담당하지 않는 것

이 문서는 현재 MVP API 메서드, 보안 보장, 아티팩트 본문 정책, 검증기 카탈로그, 적합성 fixture, 호스팅 서비스, 원격 런타임 동작, 구현 준비 상태를 정의하지 않습니다.

또한 `surface_id`, 커넥터 이름, 대시보드, 호스팅 작업 흐름, 읽기 전용 리소스를 권한으로 만들지 않습니다. 승격된 커넥터나 접점은 현재 범위와 관련 현재 담당 문서가 다시 맡아야 합니다.

## 카테고리 경계

이 카테고리의 중심 질문은 "사용자나 에이전트가 나중에 하네스와 어디에서 어떻게 상호작용할 수 있는가?"입니다. 로컬 운영자 명령, `doctor` 접점, 읽기 전용 리소스, 대시보드와 호스팅 접점, 더 넓은 커넥터 생태계, 접점 간 표시나 검증 접점을 포함합니다.

런타임 보안 주장, 아티팩트 캡처 저장소, 정책 카탈로그, 팀 생명주기는 이 문서가 담당하지 않습니다. 향후 접점이 그 영역에 의존하더라도 이 문서는 승격 전 접점 관점의 후보만 기록합니다.

## 후보 요약

| 후보 | 요약 |
|---|---|
| 향후 로컬 운영자 명령 묶음 | `harness doctor`, `harness export`, `harness conformance run` 같은 로컬 명령 접점. |
| 운영자 준비 상태와 `doctor` 접점 | 로컬 준비 상태와 진단 접점. |
| 상태 보기 새로고침과 최신성 진단 | 상태 보기 자료의 새로고침과 최신성 가시성. |
| 이후 읽기 전용 리소스 | `policy`, `evidence-manifest`, `surface`, `report`, `bundle`, `journey`, `design` 같은 읽기 전용 리소스. |
| 대시보드와 호스팅 작업 흐름 | 대시보드, 호스팅 작업 흐름, 시각화, 카드, 아티팩트 대시보드 접점. |
| 접점 간 검증 | IDE, CLI, 채팅, MCP, 호스팅 접점 사이의 검증 가시성. |
| 더 넓은 커넥터와 호스팅 런타임 | 커넥터 마켓플레이스, 호스팅 UI, 호스팅 런타임, 원격 런타임 후보. |
| 커넥터 적합성 생태계 | 커넥터 대상 호환성 주장, 마켓플레이스 신호, 보고 접점. |

## 후보 상세

<a id="future-local-operator-command-family"></a>
### 향후 로컬 운영자 명령 묶음

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: `harness connect`, `harness serve mcp`, `harness doctor`, `harness projection refresh`, `harness reconcile`, `harness recover`, `harness export`, `harness artifacts check`, `harness conformance run`을 활성화하지 않습니다.
- 승격 시 초점: 명령 접점이 승격된다면 운영 담당 문서, API 동작, 저장 효과, 적합성 점검을 함께 다룹니다.

<a id="operator-readiness-and-doctor-surfaces"></a>
### 운영자 준비 상태와 `doctor` 접점

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 준비 상태 확인이나 `doctor` 진단을 만들지 않습니다.
- 승격 시 초점: 준비 상태 접점이 승격된다면 운영 담당 문서, 보안 표현, API 동작, 적합성 점검을 함께 다룹니다.

<a id="projection-refresh-and-freshness-diagnostics"></a>
### 상태 보기 새로고침과 최신성 진단

- 상태: 이후 후보이며 현재 MVP에 포함되지 않습니다.
- 현재 MVP에 영향 없음: 현재 상태 보기 새로고침 명령, 최신성 진단, 상태 변경 상태 보기 동작을 만들지 않습니다.
- 승격 시 초점: 새로고침이나 최신성 접점이 승격된다면 상태 보기 담당 문서, API 동작, 저장 효과, 적합성 점검을 함께 다룹니다.

### 커넥터 적합성 생태계

- 현재 MVP에 영향 없음: 현재 커넥터 검증 주장, 모음 형식, 보고서, 마켓플레이스 주장을 만들지 않습니다.
- 승격 시 초점: 커넥터 대상 호환성 주장이 승격된다면 커넥터 담당 문서, 적합성 담당 문서, 보안 담당 문서, API 동작을 함께 다룹니다.

## 승격 규칙

이 파일만 고쳐서는 후보가 승격되지 않습니다. 후보는 현재 범위와 관련 현재 담당 문서가 같은 문서 전용 작업 묶음에서 갱신될 때만 활성화될 수 있습니다.

승격할 동작에 현재 담당 문서가 없다면, 그 승격 작업은 현재 API, 저장소, 보안, UI, 적합성 요구사항을 정의하기 전에 담당 문서를 만들거나 지정해야 합니다.

## 현재 범위에 영향이 없다는 점

이 문서는 현재 MVP에 영향을 주지 않습니다. 여기에 후보가 언급되어도 커넥터, 호스팅 런타임, 원격 서비스, 대시보드, UI, 로컬 명령, 읽기 전용 리소스, 접점 간 권한이 활성화되지 않습니다.
