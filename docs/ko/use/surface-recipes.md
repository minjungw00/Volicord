# 접점별 사용 레시피

에이전트가 CLI, IDE/편집기, 채팅, 로컬 MCP 같은 이름 붙은 접점에서 하네스 연결 작업을 분명하게 보여줘야 할 때 이 가이드를 사용합니다.

이 레시피는 사용자가 보는 흐름을 설명합니다. 무엇을 보여줄지, 언제 새로고침할지, 실패를 어떻게 말할지, 복사된 텍스트나 렌더링된 표시나 에이전트 기억을 권한으로 바꾸지 않으면서 상태를 어떻게 요약할지를 다룹니다. 정확한 커넥터 동작과 보안 보장은 참조 담당 문서에 둡니다.

## CLI 접점

CLI 접점은 상태를 짧고 행동 중심으로 보여줘야 합니다.

의미 있는 작업 전에는 아래를 보여줍니다.

- 현재 `Task` 또는 작업 모양
- 활성 범위와 범위 밖 항목
- 관련 경로, 명령, 동작 분류
- 대기 중인 사용자 소유 판단
- 알려진 차단 사유와 다음 안전한 행동

쓰기 가능한 작업에서는 아래를 따릅니다.

- 오래된 터미널 출력에 기대기 전에 상태를 새로고침합니다.
- 쓰기 호환성을 확인하기 전에 의도한 경로나 동작을 이름 붙입니다.
- 실행 뒤에는 무엇을 실행했는지, 무엇이 바뀌었는지, 어떤 확인이 통과하거나 실패했는지, 어떤 증거가 주장을 뒷받침하는지 요약합니다.
- 터미널 로그, 셸 기록, 로컬 경로, 복사된 요약은 맥락으로만 다루고 그 자체를 권한으로 삼지 않습니다.

현재 접점에서 명령 관찰이나 아티팩트 접근을 사용할 수 없으면 그 사실을 말하고 안전한 복구 행동을 이름 붙입니다.

## IDE/편집기 접점

IDE/편집기 접점은 모든 안전하지 않은 동작을 편집기가 막는다고 주장하지 않으면서 활성 편집 경계를 편집 흐름 가까이에 보여줘야 합니다.

보여줄 항목:

- 활성 `Task`
- 의도한 파일
- 범위 일치 또는 불일치
- 관련 있을 때 오래된 상태
- 다음 안전한 행동

편집 작업에서는 아래를 따릅니다.

- 넓은 변경이나 공개 인터페이스에 닿는 편집 전에는 작업과 범위를 새로고침합니다.
- 쓰기 전에 의도한 파일 경로를 현재 범위와 비교합니다.
- 제품, UX, 접근성, 보안, 의존성, 마이그레이션, 최종 수락, 잔여 위험 판단을 편집기 편의 프롬프트와 섞지 않습니다.
- 의미 있는 저장 뒤에는 변경 파일, 실행한 확인, 증거 공백, 닫기 차단 사유를 요약합니다.

IDE/편집기 통합은 에이전트가 절차를 지키도록 도울 수 있습니다. 하지만 보안 표현과 보장 수준은 [보안](../reference/security.md)이 담당합니다.

## 채팅 접점

채팅 접점은 작업 구체화, 판단 요청, 상태 요약, 사람이 읽기 쉬운 실패 보고에 적합합니다.

채팅 작업에서는 아래를 따릅니다.

- 사용자의 평소 요청에서 시작합니다.
- 질문하기 전에 사용할 수 있는 상태를 확인합니다.
- 사용자가 묶어서 검토하자고 하지 않는 한 막히는 사용자 소유 판단은 한 번에 하나만 묻습니다.
- 무엇이 확인되었는지, 무엇이 오래되었거나 사용할 수 없는지, 무엇이 아직 사용자 소유인지, 다음 안전한 행동이 무엇인지 요약합니다.
- "yes", "진행해", "좋아 보여"를 최종 수락, 잔여 위험 수락, 민감 동작 승인, 쓰기 호환성 전체로 한꺼번에 해석하지 않습니다.

채팅 접점이 로컬 접근, 상태, 아티팩트 가용성, 역량 지원을 확인할 수 없다면 그 사실을 직접 말하고, 다시 연결하기, 새로고침하기, 가능한 접점으로 옮기기, 동작 줄이기, 또는 명시적인 하네스 밖 계속 진행으로 안내합니다.

## 로컬 MCP 접점

로컬 MCP 접점은 상태 조회, 범위 업데이트, 쓰기 확인, 실행/증거 기록, 사용자 판단 기록, 아티팩트 스테이징, 닫기 확인을 제공하는 지원되는 하네스 메서드를 호출하는 실무 경로입니다.

로컬 MCP 작업에서는 아래를 따릅니다.

- 보호된 상태에 기대기 전에 도달 가능성을 확인합니다.
- `surface_id`를 권한이 아니라 선택자로 다룹니다.
- `state_version`, `Task` 식별자, 활성 범위, 접점 역량, 아티팩트 맥락이 바뀌면 새로고침합니다.
- 아티팩트 본문 접근은 아티팩트 스테이징, 실행/증거 기록과 분리합니다.
- 로컬 MCP를 사용할 수 없으면 담당 문서가 제공한 조건을 보고하고 다음 안전한 복구 행동을 이름 붙입니다.

로컬 MCP를 사용할 수 있다는 이유만으로 관련 담당 문서가 뒷받침하지 않는 더 강한 관찰, 예방, 격리를 암시하지 않습니다.

## 실패 보고

실패 보고는 가장 중요한 차단 사유와 그 차단을 풀 수 있는 다음 행동을 먼저 보여줘야 합니다.

차단 사유가 누구에게 속하는지 이름 붙입니다.

- 사용자 소유
- 에이전트가 풀 수 있음
- 접점/시스템 소유
- 역량 제한

접점이 시도한 일, 확인된 것, 사용할 수 없거나 오래된 것, 바뀌지 않았거나 기록되지 않은 것, 담당 문서가 제공한 코드가 있다면 그 코드, 다음 안전한 행동을 포함합니다.

## 상태 요약

상태는 짧고 판단 가능한 형태로 유지합니다.

좋은 상태 요약은 아래를 말합니다.

- 현재 `Task` 또는 작업 모양
- 활성 범위
- 가장 중요한 차단 사유
- 현재 보장 수준 또는 역량 한계
- 관련 있을 때 증거 또는 아티팩트 공백
- 관련 있을 때 닫기 가능 여부
- 다음 안전한 행동 하나

스키마 필드, 로그, 생성된 읽기용 보기, 긴 이력을 사용자에게 한꺼번에 쏟아내지 않습니다. 다음 행동이 계약 세부사항에 달려 있을 때만 정확한 참조 문서를 가져옵니다.

## 참조 담당 문서

정확한 계약은 아래 담당 경로를 사용합니다.

- [에이전트 통합 참조](../reference/agent-integration.md): 커넥터 동작, `capability_profile`, `VerifiedSurfaceContext`, 맥락 교환, 대체 경계.
- [API 메서드](../reference/api/methods.md): 지원되는 메서드 목록과 메서드 담당 경로.
- [API 오류 코드](../reference/api/error-codes.md), [API 오류 경로](../reference/api/error-routing.md): 공개 오류 코드와 복구 경로.
- [보안](../reference/security.md): 보장 표현과 비주장.
- [런타임 경계](../reference/runtime-boundaries.md): `Product Repository`, Harness Server, `Harness Runtime Home` 분리.
- [기준 범위](../reference/scope.md): 기준 범위, 프로필 조건부, 지원 범위 밖 기능 경계.

## 다음 읽기 경로

일반 에이전트 행동은 [에이전트 가이드](agent-guide.md)를 보고, 정확한 커넥터 담당 계약은 [에이전트 통합 참조](../reference/agent-integration.md)를 봅니다. 구현 담당자는 이 레시피를 API나 스키마 계약처럼 쓰지 말고 [참조 색인](../reference/README.md)에서 담당 문서 경로를 이어 갑니다.
