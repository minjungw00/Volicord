# 언어 일치 점검

이 점검은 한영 대응 문서의 의미가 바뀌거나, 한국어 산문을 고치거나, 언어 사이에서 정확히 보존해야 하는 식별자를 다룰 때 사용합니다. 어느 한 언어도 다른 언어의 보관본이나 부록이 아닙니다.

## CHK-PARITY-001: 영어와 한국어 의미 일치

담당 문서:
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [한국어 번역 가이드](../translation-guide.md)
- [doc-index.yaml](../../../doc-index.yaml)

점검:
- 의미가 바뀐 편집에서는 대응 파일을 의미 단위로 비교합니다.
- 대응 파일의 독자 목적, 규범 강도, 담당 문서 경로, 기준 범위/지원 범위 밖 경계, 사용자 판단 경계, 보안 보장 수준이 같은지 확인합니다.
- 한국어는 줄 단위 번역이 아니라 자연스러운 구조를 허용합니다.

실패 조건:
- 한 언어가 의미 변경을 빠뜨립니다.
- 한 언어가 대응 파일보다 규칙을 강하게 하거나 약하게 하거나 다른 담당 문서로 보냅니다.

수정 방향:
- 같은 문서 작업 묶음에서 두 언어를 함께 고칩니다.
- 의미 일치를 유지하면서 한국어는 자연스럽게 다시 씁니다.

## CHK-PARITY-002: 정확한 식별자 보존

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [번역 가이드](../translation-guide.md)

점검:
- 정확한 식별자가 두 언어에서 그대로 남아 있는지 확인합니다.
- 파일 경로, 앵커, `doc_id` 값, API 메서드, 스키마 필드, enum 값, 테이블 이름, validator ID, 오류 코드, 제품 라벨은 문맥상 명확성이나 검색성이 필요할 때 백틱으로 감쌉니다.

실패 조건:
- 정확한 식별자를 번역, 지역화, 재서식화하거나 의미가 달라지는 표시 라벨처럼 씁니다.

수정 방향:
- 정확한 식별자를 복원합니다.
- 필요하면 식별자 옆에 평이한 설명을 덧붙입니다.

## CHK-PARITY-003: 한국어 구조 보존

담당 문서:
- [한국어 번역 가이드](../translation-guide.md)
- [한국어 작성 가이드](../authoring-guide.md)
- [영어 작성 가이드](../../../en/maintain/authoring-guide.md)

점검:
- 한국어 참조 문서를 고쳤다면 조건, 결과, 예외, 비주장, 담당 문서 링크, 닫기 준비 상태의 결과를 의미 단위로 비교합니다.
- 한국어 문장의 줄 수와 순서는 달라도 되지만, 중요한 주의 사항과 담당 경계가 보이는지 확인합니다.
- 촘촘한 한국어 문단이 조건, 예외, 비주장을 합쳐 숨기지 않았는지 확인합니다.

실패 조건:
- 한국어 문구가 넓은 주제는 보존하지만 조건/결과/예외나 비주장 구조를 하나로 뭉갭니다.
- 담당 경계, 기준 범위/지원 범위 밖 경계, 보안 비주장, 닫기 준비 상태의 결과가 대응 의미 단위보다 찾기 어려워집니다.

수정 방향:
- 의미 단위가 드러나도록 자연스러운 한국어 문단이나 글머리표로 나눕니다.
- 정확한 식별자를 그대로 두고, 줄 단위 번역을 강제하지 않으면서 의미 일치를 보존합니다.

## CHK-PARITY-004: 한국어 저장소 구조

담당 문서:
- [참조 색인](../../reference/README.md)과 `doc-index.yaml`에서 고른 적용되는 대응 저장소 담당 문서
- [번역 가이드](../translation-guide.md)

점검:
- 한국어 저장소 참조 문서를 고쳤다면 대응 영어 저장소 원천 문서를 의미 단위로 비교합니다.
- 조건, 효과, 예외, 비주장, 담당 문서 링크가 한국어에서도 분리되어 보이는지 확인합니다.
- 촘촘한 한국어 문단이 저장소 규칙을 합쳐 조건, 예외, 비주장을 숨기지 않는지 확인합니다.

실패 조건:
- 중요한 저장소 조건, 효과, 예외, 비주장이 촘촘한 한국어 문단 안에 뭉쳐 있습니다.
- 넓은 주제는 맞지만 저장소 경계가 대응 영어 의미 단위보다 검토하기 어려워집니다.

수정 방향:
- 의미 단위가 드러나도록 자연스러운 문단, 글머리표, 표로 다시 씁니다.
- 정확한 식별자는 그대로 두고, 유지보수 지침에 계약 세부사항을 반복하지 말고 저장소 담당 문서로 연결합니다.

## CHK-PARITY-005: 한국어 사용자 문서 가독성

담당 문서:
- [한국어 작성 가이드](../authoring-guide.md)
- [한국어 번역 가이드](../translation-guide.md)

점검:
- 한국어 사용자 문서가 자연스러운 한국어 기술 문서인지, 한국어 개념을 먼저 두는지, 용어가 일관되는지 확인합니다.
- 정확한 식별자는 검색 가능하게 보존하되 일반 표시 라벨처럼 드러내지 않는지 확인합니다.
- 일반 영어 명사와 명사구는 정확한 식별자, 의도적 제품 라벨, 자연스러운 기술 차용어가 아니라면 한국어로 옮겼는지 확인합니다.
- 영어 어순을 그대로 비추는 것보다 자연스러운 한국어 구조가 같은 의미를 더 분명히 전달한다면 한국어 구조로 다시 썼는지 확인합니다.

실패 조건:
- 한국어 문장이 영어 어순을 비추거나, 피할 수 있는 영어 명사구를 남기거나, 내부 식별자 뒤에 독자 행동을 숨깁니다.
- 대응 영어 문장에 있었다는 이유만으로 일반 영어 명사를 한국어 산문에 남깁니다.
- 식별자나 의도적 라벨이 아닌 영어 명사 묶음을 화면에 보이는 산문으로 그대로 남깁니다.

수정 방향:
- 식별자와 의미 일치를 보존하면서 자연스러운 한국어로 다시 씁니다.

## CHK-PARITY-006: 대응 제목과 읽기 구조

담당 문서:
- [번역 가이드](../translation-guide.md)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [작성 가이드](../authoring-guide.md)

점검:
- 제목이 바뀌면 대응 영어/한국어 파일을 제목 의미와 읽기 구조 기준으로 비교합니다.
- 섹션을 추가, 삭제, 분리, 병합했다면 대응 파일이 같은 의미 단위를 대응 제목이나 가까운 문맥으로 드러내는지 확인합니다.
- 두 언어가 같은 주요 의미 단위, 담당 경로, 주의 사항, 예외, 체크리스트 범위를 드러내는지 확인합니다.
- 한국어의 보이는 제목은 자연스럽게 두고, 안정적인 영어 앵커가 필요하면 숨김 앵커를 쓰는지 확인합니다.

실패 조건:
- 한 언어의 제목 변경이 대응 파일보다 의미 단위를 추가하거나, 제거하거나, 약하게 하거나, 다른 담당 문서로 보냅니다.
- 섹션 구조 때문에 한 언어에서는 규칙을 쉽게 찾을 수 있지만 다른 언어에서는 의미 있게 찾기 어려워집니다.
- 앵커를 맞추려고 한국어 제목이 영어 표현이나 어순을 보존합니다.

수정 방향:
- 같은 문서 작업 묶음에서 대응 제목과 주변 맥락을 함께 갱신합니다.
- 안정적인 앵커가 필요하면 숨김 앵커로 보존하고, 보이는 한국어 제목은 자연스럽게 둡니다.
