# 용어 점검

이 점검은 제품 용어, 한국어 산문 용어, 혼합어 표현, 식별자 설명, 문서 경로 안내 용어, 닫기 준비 상태 표현, 접근/보안 표현이 바뀔 때 사용합니다. 유지보수 용어 통제는 용어 지도가 담당하고, 제품 계약은 각 참조 담당 문서에 남습니다.

## CHK-TERM-001: 닫기 준비 상태 용어

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [용어집](../../reference/glossary.md)
- [번역 가이드](../translation-guide.md)

점검:
- 한국어 참조 문서는 "닫기 준비 상태"를 씁니다.
- 한국어 사용자 문서는 "닫기 가능 여부"를 쓸 수 있습니다.
- 한국어 산문은 "close 가능성 평가"나 "닫기 가능성 평가"를 쓰지 않습니다. 단, 금지 표현 목록이나 인용된 과거 예시는 예외입니다.

실패 조건:
- 금지된 혼합 표현이 금지 표현 목록이나 인용된 과거 예시 밖에 나타납니다.
- 사용자 문서 표현과 참조 문서 표현을 바꾸어 독자 의미가 달라집니다.

수정 방향:
- 용어 지도에 맞게 표현을 바꿉니다.
- 용어 지도가 부족하면 새 표현을 퍼뜨리기 전에 용어 담당 문서와 대응 가이드를 함께 고칩니다.

## CHK-TERM-002: 용어 변동

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [용어집](../../reference/glossary.md)
- [번역 가이드](../translation-guide.md)

점검:
- 바뀐 산문에서 새 제품 용어, 혼합어 한국어, 기존 개념의 다른 표기를 찾습니다.
- 오래 유지될 새 용어가 용어집, 용어 지도, 또는 알맞은 참조 담당 문서에 있는지 확인합니다.
- 한국어 문장이 식별자, 의도적 제품 라벨, 자연스러운 기술 차용어가 아닌 일반 영어 명사구를 한국어로 옮겼는지 확인합니다.

실패 조건:
- 같은 개념이 담당 문서의 구분 없이 여러 표현으로 나타납니다.
- 식별자, 의도적 제품 라벨, 자연스러운 기술 차용어가 아닌 영어 명사구가 한국어 문장에 남습니다.

수정 방향:
- 담당 용어 문서에 맞게 표현을 정리합니다.
- 새 구분이 의도적일 때만 담당 용어를 추가하거나 고칩니다.

## CHK-TERM-003: `complete` 의도 값 모호성

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [용어집](../../reference/glossary.md)
- [API 값 집합](../../reference/api/schema-value-sets.md)
- [API 메서드](../../reference/api/methods.md)

점검:
- `complete`는 식별자 또는 enum 값일 때만 보존합니다.
- 영어가 full 또는 entire 의미일 때 한국어 산문에 `complete`를 남기지 않습니다.
- 한국어 산문이 "complete 닫기 준비 상태 순서" 같은 표현을 쓰지 않는지 확인합니다.
- 영어 문서에서는 enum 값과 헷갈릴 수 있는 "Complete ..." 대신 필요한 경우 "Full ..."을 씁니다.

실패 조건:
- 영어가 full 또는 entire 의미인데 한국어 문장이 `complete`를 보존합니다.
- 제목이 실제로는 `intent=complete`가 아닌데 `complete`를 그 값처럼 보이게 합니다.

수정 방향:
- "전체", "전체 평가", 또는 문맥에 맞는 자연스러운 한국어 표현을 씁니다.
- 영어에서는 enum 값 모호성을 피해야 할 때 "Full ..."을 우선합니다.

## CHK-TERM-004: 접점, 접근, 보안 표현

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [보안](../../reference/security.md)
- [에이전트 통합](../../reference/agent-integration.md)
- [번역 가이드](../translation-guide.md)

점검:
- `surface_id`, 접점, 커넥터, 역량, 접근 등급 표현이 담당 문서 근거 없이 권한, 승인, 바인딩 증거를 뜻하지 않는지 확인합니다.
- 접근 관련 용어가 문서 지침과 런타임 강제를 구분하는지 확인합니다.
- 협력형, 탐지형, 예방형, `guard`, `freeze`, `careful mode`, 샌드박스, 권한, 차단, 변조 방지, 격리, 역량 표현이 담당 문서 근거 안에 있는지 확인합니다.

실패 조건:
- 접점이나 접근 용어를 담당 문서 근거 없이 권한 증거, 사용자 판단, `Write Authorization`, 보안 격리, 런타임 강제처럼 씁니다.
- 경로 문서, 예시, 지원 범위 밖 자료에 언급되었다는 이유만으로 더 강한 격리, 샌드박싱, 권한 강제, 변조 불가능성을 지원된다고 설명합니다.

수정 방향:
- 문맥에 맞게 식별, 경로 안내, 문서화된 지침으로 다시 씁니다.
- 보장 의미는 보안, 커넥터 맥락은 에이전트 통합, 지원 여부는 필요한 경우 범위로 연결합니다.

## CHK-TERM-005: 용어 지도 정렬

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [영어 번역 가이드](../../../en/maintain/translation-guide.md)
- [한국어 번역 가이드](../translation-guide.md)

점검:
- 바뀐 용어 지침을 `docs/terminology-map.yaml`과 대조합니다.
- `owner_documents` 대상이 집중 담당 문서가 있을 때 넓은 색인이나 문서 묶음 페이지가 아니라 그 문서를 가리키는지 확인합니다.
- 용어집의 담당 문서 링크도 같은 집중 담당 문서 원칙을 따르는지 확인합니다.
- 가이드의 금지 혼합어 예시가 흐릿한 설명이 아니라 실제 문자열인지 확인합니다.
- 새 금지 표현이 용어 지도와 양쪽 번역 가이드에 함께 들어 있는지 확인합니다.

실패 조건:
- 가이드와 용어 지도가 어긋납니다.
- 용어 지도나 용어집의 담당 대상이 용어 의미, 값 집합, API 관심사, 저장소 관심사, 표시 문구를 이미 담당하는 집중 문서 대신 색인을 가리킵니다.
- 한국어 가이드가 금지 패턴을 설명만 하고, "artifact를 저장한다"처럼 검색 가능한 실제 문자열을 쓰지 않습니다.

수정 방향:
- 같은 문서 작업 묶음에서 용어 지도와 양쪽 가이드를 맞춥니다.
- 담당 링크를 집중 담당 문서로 바꿉니다. 색인은 탐색 개념 자체이거나 담당 문서 공백을 명시할 때만 사용합니다.
- 흐릿한 자리표시자는 검색 가능한 실제 예시 문자열로 바꿉니다.

## CHK-TERM-006: `active`와 지원/적용 표현 구분

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [용어집](../../reference/glossary.md)
- [Core 모델](../../reference/core-model.md)
- [에이전트 통합](../../reference/agent-integration.md)

점검:
- 바뀐 산문에서 `active`를 검색합니다.
- `active`가 런타임 또는 현재 적용 상태, 정확한 식별자, 정확한 상태 값, active scope, active Change Unit, active surface context를 말할 때만 쓰였는지 확인합니다.
- 지원되는 계약, 지원되는 API 메서드, 지원되는 값, 유지되는 문서, 담당 경로는 `active`가 아니라 "지원되는", "적용되는", "유지되는", "현재" 같은 문맥에 맞는 표현을 쓰는지 확인합니다.

실패 조건:
- 적용되는 담당 경로, 지원되는 계약, 지원되는 API 메서드, 유지되는 참조 문서에 `active` 수식어를 붙입니다.
- 한국어 산문이 문서 계약이나 담당 경로의 `active`를 "활성"으로 옮깁니다.

수정 방향:
- 문맥에 맞게 "적용되는 담당 경로", "지원되는 API 메서드", "지원되는 값", "유지되는 문서", "현재 상태"로 바꿉니다.
- `active`는 정확한 식별자, 상태 값, 현재 적용 런타임/세션 상태를 말할 때만 남깁니다.

## CHK-TERM-007: 더 이상 쓰지 않거나 지원되지 않는 개념 이름

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [범위](../../reference/scope.md)
- [참조 색인](../../reference/README.md)
- [작성 가이드](../authoring-guide.md)
- [템플릿 본문](../../reference/template-bodies.md)

점검:
- 유지되는 참조 문서, 용어집, `docs/terminology-map.yaml`, `doc-index.yaml`, 표시 문구 담당 문서, 바뀐 메타데이터에서 더 이상 쓰지 않거나, 삭제되었거나, 지원되지 않는 개념 이름을 검색합니다.
- 영어 개념 라벨을 제거했다면 한국어 산문에서 같은 제거된 개념을 보존하는 바꿔 쓴 표현, 번역 표현, 혼합어 변형, 표 행, 목록 항목, 제목을 찾습니다.
- 의미 담당 문서가 정확한 이름을 여전히 필요로 하거나, 유지보수/용어 담당 문서가 검색 가능한 금지 표현으로 의도적으로 보관하는 경우에만 지원되지 않는 기능 이름을 남기는지 확인합니다.
- 참조 담당 문서가 오래된 이름을 예시로 보존하기보다 안정적인 범주, 담당 문서 공백, 지원 범위 밖 기능 묶음으로 설명하는지 확인합니다.
- 부정 예시가 제거된 이름을 지원 개념, 담당 경로, 저장소 기록 묶음, 렌더링 본문 묶음처럼 보이게 하지 않는지 확인합니다.

실패 조건:
- 용어집 항목, 용어 지도 항목, 메타데이터 경로, 참조 문서, 표시 문구 담당 문서가 제거되었거나 지원되지 않는 개념 이름을 "지원하지 않는다"라고 말하기 위해서만 보존합니다.
- 제거된 영어 라벨은 문자 그대로 없어졌지만 한국어 바꿔 쓰기나 번역 표현이 그 제거된 개념을 현재 쓰는 개념처럼 보이게 합니다.
- 부정 예시 때문에 검색이 오래된 이름을 현재 적용 계약, 지원 기능, 담당 경로처럼 취급할 수 있습니다.
- 표시 문구 담당 문서나 저장소 관련 메모가 지원되지 않는 렌더링 본문 또는 저장소형 묶음 이름을 남겨 공식 개념처럼 검색되게 합니다.

수정 방향:
- 오래된 이름을 제거하거나 안정적인 범주와 적용되는 담당 문서 링크로 바꿉니다.
- 유지보수자가 검색해야 하는 금지 용어라면 용어 지도와 번역 가이드로 옮깁니다.
- 지원되지 않는 표시 문구나 저장소처럼 보이는 이름은 용어 담당 문서가 금지 용어로 의도적으로 보존하는 경우가 아니면 제거합니다.

## CHK-TERM-008: 문서 경로 안내 용어는 문서 용도로만 사용

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [용어집](../../reference/glossary.md)
- [작성 가이드](../authoring-guide.md)

점검:
- 바뀐 산문에서 `applicable owner path`, 담당 경로, 담당 대상, 경로 메타데이터, 담당 문서 공백 같은 문서 경로 안내 용어를 찾습니다.
- 이 용어들이 문서 탐색, 작성, 검색, 메타데이터만 설명하는지 확인합니다.
- 담당 경로 용어가 제품 동작, 저장 지속 조건, API 지원 여부, 증거 권한, 닫기 준비 상태, 사용자에게 보이는 표시의 행위 주어가 되지 않는지 확인합니다.
- 제품 동작, 저장 지속 조건, 런타임 상태, 증거 권한, 닫기 준비 상태, API 지원 여부처럼 쓰이지 않는지 확인합니다.

실패 조건:
- 문서 경로 안내 용어가 지속 제품 필드, 런타임 상태, API 값, 저장소 기록, 지원 보장, 닫기 준비 상태 결과처럼 쓰입니다.
- 담당 경로, 경로, 메타데이터 항목이 제품 동작을 수행하거나, 차단하거나, 허용하거나, 저장하거나, 검증하거나, 수락하거나, 표시하거나, 권한을 부여하는 것처럼 문장이 쓰입니다.
- 담당 경로가 적용된다는 이유로 제품 동작이 지원된다고 설명하고, 범위와 의미 담당 문서의 지원 정의를 확인하지 않습니다.

수정 방향:
- 문서 경로 안내 표현을 문서 경로 안내나 메타데이터로 다시 씁니다.
- 담당 문서나 담당 계약은 정의의 출처로만 이름 붙이고, 제품 동작, 저장 지속 조건, 런타임 상태, API 지원 여부는 집중 제품 담당 문서로 보냅니다.

## CHK-TERM-009: 한국어 차단 사유 용어

담당 문서:
- [용어 지도](../../../terminology-map.yaml)
- [용어집](../../reference/glossary.md)
- [번역 가이드](../translation-guide.md)

점검:
- 닫기와 관련된 차단 사유를 산문으로 설명할 때는 "닫기 차단 사유"를 쓰는지 확인합니다.
- 차단 사유 범주를 산문으로 설명할 때는 "차단 사유 범주"를 쓰고, 정확한 필드 이름을 말할 때는 `CloseReadinessBlocker.category`를 보존하는지 확인합니다.
- 차단 사유 처리 경로를 산문으로 설명할 때는 "차단 사유 처리 경로"를 쓰고, 정확한 담당 경로나 식별자를 말할 때는 그대로 보존하는지 확인합니다.
- "close blocker", "blocker category", "blocker 처리 경로", "blocker 라우팅" 같은 변형은 금지 표현 목록이나 인용된 예시에만 있는지 확인합니다.

실패 조건:
- 한국어 문서가 같은 개념에 대해 용어 담당 문서가 구분하지 않은 차단 사유 용어 변형을 섞어 씁니다.
- 용어 지도가 한국어 표현을 제공하는데도 식별자가 아닌 영어식 차단 사유 표현이 한국어 산문에 남습니다.
- 정확한 스키마나 필드 식별자를 번역합니다.
