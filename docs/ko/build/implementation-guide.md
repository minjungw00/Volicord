# 구현 가이드

이 가이드는 하네스 제품 범위에서 런타임 동작을 정의하는 담당 문서까지 이어지는 안정적인 구현 경로를 설명합니다.

제품 범위, API 동작, 스키마, 저장 효과, 보안 보장, 커넥터 동작은 이 문서가 정의하지 않습니다. 해당 계약은 참조 담당 문서에 둡니다.

## 담당 문서 우선 원칙

구현은 경로 요약이 아니라 담당 문서를 기준으로 잡습니다.

구현 단위를 계획할 때는 아래 순서로 확인합니다.

1. [범위](../reference/scope.md)에서 활성 기능 경계를 확인합니다.
2. [참조 색인](../reference/README.md)에서 정확한 계약 담당 문서를 찾습니다.
3. 공개 메서드 목록과 메서드 담당 문서 경로는 [API 메서드](../reference/api/methods.md)를 사용합니다.
4. 여러 경계를 건너는 동작은 메서드 담당 문서, 스키마 담당 문서, 저장소 담당 문서, 보안 담당 문서를 함께 확인합니다.
5. 사용자 소유 판단, 증거, 검증 기대치, 수락, 닫기 준비 상태, 잔여 위험은 서로 다른 권한 개념으로 유지합니다.

## 첫 런타임 단위

가장 작은 유용한 하네스 런타임 단위는 평범한 사용자 작업 하나를 아래 흐름으로 다룹니다.

- 자연어 접수
- 활성 범위 확인
- 상태 검토
- 쓰기 호환성 확인
- 필요한 경우 아티팩트 스테이징
- 실행과 증거 기록
- 필요한 사용자 소유 판단
- 닫기 준비 상태 평가

이 단위는 현재 계약만 사용합니다. 범위 밖 기능이나 예약된 기능은 [범위](../reference/scope.md)와 영향받는 담당 문서가 활성 동작으로 설명한 뒤에만 구현 입력이 됩니다.

## 구현 규율

구현 작업은 의존하는 담당 문서를 이름 붙여야 합니다. 어떤 구현 단위에 담당 문서가 정의하지 않은 동작이 필요하다면 추론한 규칙을 코드에 넣기보다 담당 문서를 먼저 갱신합니다.

런타임 상태, 생성된 아티팩트, 증거 기록, QA 기록, 수락 기록, 닫기 기록, 잔여 위험 기록, 픽스처 출력, 제품 구현 파일은 이 문서 트리에 저장하지 않습니다.

이 문서의 경로 허용 목록, 경로 표, 작업 묶음 경계는 문서 세트를 관리하기 위한 편집 통제입니다. 하네스 런타임 권한, 쓰기 권한 부여, 샌드박스 보장, 집행 증명이 아닙니다.

## 예시 스모크 경로

대표 첫 스모크 경로는 API 예시 시나리오를 사용할 수 있습니다.

- 계정 데이터 내보내기 전에 명시적 확인 단계를 추가하는 작업을 시작합니다.
- 계정 삭제 동작이 범위 밖임을 확인합니다.
- 호환되는 제품 파일 쓰기를 준비합니다.
- 아티팩트 담당 문서가 허용할 때 대표 테스트 출력을 아티팩트로 스테이징하거나 연결합니다.
- 실행과 압축된 증거 요약을 기록합니다.
- 필요한 사용자 소유 판단을 요청합니다.
- `harness.close_task`로 닫기 준비 상태를 평가합니다.

스모크 경로는 계획 형태입니다. 정확한 요청, 응답, 저장 효과, 오류 동작, 닫기 차단 사유는 관련 참조 담당 문서가 정의합니다.
