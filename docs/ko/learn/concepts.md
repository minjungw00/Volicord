# 핵심 개념

## 여기서 시작하기

[개요](overview.md)와 [하나의 작업](one-task.md)을 읽은 뒤, 상태 요약에서 하네스 어휘가 보이기 시작할 때 이 문서를 봅니다.

이 문서는 처음 읽는 사람에게 필요한 최소 개념만 다룹니다. 이 저장소는 문서 전용입니다. 여기서 설명하는 향후 런타임 기록이 이미 이 저장소에 있다는 뜻은 아닙니다.

## 일곱 가지 질문

하네스는 일곱 가지 평소 질문으로 읽으면 쉽습니다.

| 질문 | 쉬운 개념 |
|---|---|
| 무엇을 하려는가? | 작업 |
| 무엇이 바뀔 수 있고, 무엇은 범위 밖인가? | 범위 |
| 어떤 결과면 충분한가? | 완료 기준 |
| 사용자가 무엇을 결정, 수락, 면제, 보류해야 하는가? | 사용자 소유 판단 |
| 무엇이 주장을 뒷받침하는가? | 증거 |
| 무엇을 확인했고, 무엇이 아직 필요한가? | 확인, 검증, 수동 QA |
| 정직하게 끝낼 수 있는가, 무엇이 아직 막는가? | 닫기 가능 여부 |

사용자는 이 말을 정확히 외울 필요가 없습니다. 평소 말이면 충분합니다.

```text
이 계획을 구현할 수 있을 만큼 구체화해줘.
이 기능을 구현 가능한 계획으로 구체화해줘.
내가 결정해야 할 부분을 먼저 물어봐줘.
변경 전에 어떤 파일을 바꿀지 확인해줘.
완료했다고 말하기 전에 증거와 남은 위험을 보여줘.
```

## 세 가지 작업 모양

하네스는 보통 세 가지 작업 모양으로 느껴져야 합니다.

| 모양 | 어떤 느낌인가 | 경계 |
|---|---|---|
| 조언/읽기 전용 작업 | 설명, 계획, 비교, 조사, 추천. | 조언은 작업 방향을 잡을 수 있지만 제품 쓰기나 위험 수락을 뜻하지 않습니다. |
| 작은 직접 변경 | 범위가 좁은 오탈자, 문구, 작은 수정. | 범위가 유지되는 동안 가볍게 처리합니다. 의미, 위험, UX, 민감 동작, 공유 계약 영향이 보이면 멈춥니다. |
| 추적되는 작업 | 의미 있는 범위, 완료 기준, 사용자 선택, 증거, QA, 검증, 수락, 잔여 위험이 있는 작업. | 범위, 판단, 증거, 확인, 남은 위험, 닫기 차단 사유를 보이게 유지합니다. |

사용자는 이 모양의 이름을 말할 필요가 없습니다. 에이전트가 요청에서 추론하고, 모양이 바뀌면 설명해야 합니다. 충분한 정보가 있으면 추적되는 작업은 계획에 머물지 말고 다음 안전한 구현 조각으로 나아가야 합니다.

## 대체 금지 규칙

아래 규칙 때문에 개념을 나누는 일이 중요합니다.

| 규칙 | 뜻 |
|---|---|
| 대화는 상태가 아닙니다. | 대화는 조율할 수 있지만 오래 남는 운영 기록은 아닙니다. |
| 읽기용 요약은 상태가 아닙니다. | 보고서는 상태를 보여줄 수 있지만, 보고서 문장을 고친다고 향후 하네스 기록이 바뀌지는 않습니다. |
| 도구 출력은 사용자 판단이 아닙니다. | 로그, diff, 테스트, 스크린샷, connector 응답은 선택에 도움을 줄 수 있지만 선택을 대신하지 않습니다. |
| 민감 단계 승인은 최종 수락이 아닙니다. | 이름 붙은 행동을 승인해도 완료 결과를 최종 수락한 것은 아닙니다. |
| 증거는 검증이 아닙니다. | 증거는 주장을 뒷받침합니다. 검증은 무엇을 어떻게 확인했는지 말합니다. |
| 테스트 통과는 수동 QA가 아닙니다. | 자동 확인은 문구, 접근성, 시각 품질, 사용자 경험을 증명하지 않습니다. |
| 잔여 위험 표시는 잔여 위험 수락이 아닙니다. | 남은 위험을 보여줬다고 사용자가 수락한 것은 아닙니다. |
| "진행해"나 "좋아 보여"가 모든 답은 아닙니다. | 막연한 표현을 관련 없는 판단까지 해결한 답으로 늘리면 안 됩니다. |

## 나중에 볼 수 있는 라벨

내부 라벨은 찾아보기 도구입니다. 처음 이해해야 하는 개념이 아닙니다. Learn과 Use 문서는 쉬운 상황을 먼저 설명해야 합니다.

| 라벨 | 쉬운 뜻 |
|---|---|
| Task | 작업 하나를 오래 남기는 단위. |
| Change Unit | 제품 쓰기 범위. 그 자체로 쓰기를 승인하지는 않습니다. |
| Decision Packet | 특정 사용자 소유 판단을 자세히 보여주는 형식. |
| Approval | 이름 붙은 민감 동작에 대한 승인. 최종 수락이 아닙니다. |
| Write Authorization | 쓰기 전 범위 확인 뒤에 있는 내부 기록/확인. OS 권한이나 샌드박스가 아닙니다. |
| Evidence Manifest | 자세한 증거 목록. |
| Verification / Manual QA / Acceptance / Residual Risk | 하나의 "완료"로 합치면 안 되는 별도 기록이나 판단. |
| Projection | 하네스 기록에서 나온 읽기용 보기. 그 자체가 권한은 아닙니다. |

정확한 정의, API 형태, 저장 규칙, 상태 전이는 Learn이 아니라 Reference 문서에 있습니다.
