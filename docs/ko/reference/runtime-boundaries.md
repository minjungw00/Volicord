# 런타임 경계 참조

이 문서에서는 `Product Repository`를 제품 저장소, `Harness Server`를 하네스 서버, `Harness Runtime Home`을 런타임 홈이라고 부릅니다. 이 문서는 제품 저장소, 하네스 서버 또는 설치 위치, 런타임 홈 사이의 경계를 담당합니다.

조건:

- 제품 파일, 하네스 서버 설치 파일, 하네스 런타임 데이터가 어디에 있는지에 따라 주장이 달라질 때 이 문서를 사용합니다.
- 이 저장소는 하네스 서버를 위한 참조 문서로만 봅니다.

주장할 수 있는 것:

- 이 저장소의 Markdown 파일은 하네스 규칙을 설명할 수 있습니다.
- 제품 저장소, 하네스 서버 또는 설치 위치, 런타임 홈은 서로 다른 위치입니다.

주장하면 안 되는 것:

- 문서 파일이 하네스 서버/런타임 구현 자료, 런타임 홈 데이터, 생성된 상태 보기 시스템, 적합성 실행기, 런타임 상태, 런타임 데이터라는 주장.
- 문서 파일이 런타임 상태, 하네스 기록, 생성된 아티팩트, 상태 보기, 수락 기록, 증거 기록, 운영 파일이라는 주장.

담당 문서 링크:

- [보안](security.md): 보안 보장 의미와 비주장.
- [저장소 기록](storage-records.md), [저장 효과](storage-effects.md), [아티팩트 저장소](storage-artifacts.md): 저장소 세부사항.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 제품 저장소, 하네스 서버 또는 설치 위치, 런타임 홈의 분리
- 제품 저장소 파일이 하네스 권한을 만들지 않는다는 규칙
- 생성된 표시, 대화 텍스트, 커넥터 설명, 에이전트 기억이 하네스 권한을 만들지 않는다는 규칙
- 서버 설치 위치와 런타임 데이터 위치를 구분하는 규칙
- 런타임 홈이 자동으로 보안 경계가 아니라는 경계

이 문서는 담당하지 않습니다.

- 저장소 기록, 효과, 아티팩트, 버전 관리, 잠금, 마이그레이션: [참조 색인](README.md)의 저장소 담당 문서
- 공개 API 스키마나 메서드 동작: [참조 색인](README.md)의 API 담당 문서
- 보안 보장 의미나 자세한 보안 비주장: [보안](security.md)
- 상태 보기 권한: [상태 보기 권한 참조](projection-and-templates.md)

## 제품 저장소

제품 저장소는 사용자의 프로젝트 작업 공간입니다.

조건:

- 제품 파일은 하네스 확인이나 사용자 소유 판단의 입력이 될 수 있습니다.
- 프로젝트 안 하네스 메타데이터는 저장소/런타임 담당 문서가 그 의미를 정의할 때만 허용됩니다.

주장할 수 있는 것:

- 제품 저장소 파일은 사용자 작업 공간 파일입니다.
- 담당 문서가 정의한 프로젝트 안 하네스 메타데이터가 있을 수 있습니다.

주장하면 안 되는 것:

- 제품 저장소 내용이 하네스 상태라는 주장.
- 제품 저장소 내용이 생성된 하네스 출력이라는 주장.
- 제품 저장소 내용이 하네스 권한 증거라는 주장.
- 제품 저장소가 자동으로 런타임 홈이라는 주장.
- 프로젝트 안 하네스 메타데이터가 있다는 이유로 일반 제품 파일이 하네스 기록이 된다는 주장.

담당 문서 링크:

- [저장 효과](storage-effects.md): 제품 파일과 하네스 기록의 저장 효과.
- [Core 모델](core-model.md): 사용자 소유 판단 경계.
- [보안](security.md): 보안 권한 비주장.

## 하네스 서버 또는 설치 위치

하네스 서버는 하네스 기록과 API 동작을 중재합니다.

조건:

- 서버 설치 위치는 서버 코드, 패키지, 설정, 애플리케이션 리소스가 놓일 수 있는 곳입니다.
- 어떤 디렉터리가 런타임 홈이 되려면 저장소/런타임 담당 문서가 그렇게 정의해야 합니다.

주장할 수 있는 것:

- 하네스 서버는 하네스 기록과 API 동작을 중재합니다.
- 하네스 서버 설치 위치와 런타임 데이터 위치는 서로 다를 수 있습니다.

주장하면 안 되는 것:

- 설치 위치와 런타임 데이터 위치가 기본적으로 같다는 주장.
- 어떤 디렉터리에서 하네스 서버를 설치하거나 실행하면 그 디렉터리가 런타임 홈이 된다는 주장.
- 문서 파일이 실행 가능한 서버 자료라는 주장.
- 문서 편집이 서버 코드를 만들거나, 런타임 동작을 시작하거나, 제품/런타임 쓰기를 승인한다는 주장.

담당 문서 링크:

- [API 메서드](api/methods.md): API 메서드 동작 경로.
- [저장소 기록](storage-records.md): 런타임 데이터 기록 배치.
- [저장 효과](storage-effects.md): 제품/런타임 쓰기 효과.

## 런타임 홈

런타임 홈은 사용자별 또는 설치별 운영 데이터 공간입니다.

조건:

- 저장소/런타임 담당 문서가 런타임 홈에 속하는 운영 데이터를 정의합니다.

포함될 수 있는 것:

- 하네스 소유 기록.
- 로컬 저장소 메타데이터.
- 스테이징되었거나 지속된 아티팩트 데이터.
- 잠금, 마이그레이션, 관련 진단.

주장할 수 있는 것:

- 런타임 홈은 저장소/런타임 담당 문서가 데이터와 검증 규칙을 정의할 때 하네스 운영 데이터를 담을 수 있습니다.

주장하면 안 되는 것:

- 런타임 홈이 제품 저장소라는 주장.
- 런타임 홈이 자동으로 보안 경계라는 주장.
- 이 문서 저장소가 런타임 홈이라는 주장.

- [저장소 기록](storage-records.md), [저장 효과](storage-effects.md), [아티팩트 저장소](storage-artifacts.md), [저장소 버전 관리](storage-versioning.md): 저장소와 런타임 데이터 세부사항.

주장할 수 있는 것:
- 하네스 동작을 위한 원천 문서를 둘 수 있습니다.

주장하면 안 되는 것:
- 문서 저장소 저장 범위가 런타임 상태라는 주장.
- 문서 저장소 저장 범위가 실행 가능한 서버 자료라는 주장.
- 문서 저장소 저장 범위가 생성된 상태 보기를 담는다는 주장.
- 문서 저장소 저장 범위가 증거, QA, 수락, 닫기 기록, 적합성 결과를 담는다는 주장.

담당 문서 링크:
- [구현 가이드](../build/implementation-guide.md): 유지보수자 구현 경로.
- [보안](security.md): 보안 비주장.

## 추론하면 안 되는 것

조건:

- 독자가 문서, 제품 저장소 텍스트, 생성된 Markdown, 렌더링된 표시, 커넥터 설명, 대화 텍스트, 에이전트 기억에서 하네스 개념을 봅니다.
- 독자가 복사된 `surface_id`, 표시된 `ArtifactRef`, 렌더링된 상태 보기, 설치 디렉터리, 로컬 디렉터리 이름을 봅니다.

주장하거나 추론하면 안 되는 것:

- 이 저장소에 동작하는 하네스 서버, 런타임 홈, 런타임 데이터가 있다는 주장.
- 문서 파일이 하네스 기록이나 생성된 운영 파일이라는 주장.
- 담당 문서가 정한 런타임 설정이 없는 한 제품 저장소가 런타임 홈이라는 주장.
- 서버 설치 디렉터리가 런타임 데이터 디렉터리라는 주장.
- 런타임 홈이 보안 경계라는 주장.
- 제품 저장소 텍스트, 생성된 Markdown, 렌더링된 표시, 렌더링된 상태 보기가 하네스 권한을 만든다는 주장.
- 대화 텍스트, 커넥터 설명, 에이전트 기억이 하네스 권한을 만든다는 주장.
- 복사된 `surface_id` 값이나 표시된 `ArtifactRef` 값이 하네스 권한을 만든다는 주장.
