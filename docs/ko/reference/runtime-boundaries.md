# 런타임 경계 참조

이 문서는 Product Repository(제품 저장소), Harness Server(하네스 서버) 또는 설치 위치, Harness Runtime Home(하네스 런타임 홈) 사이의 경계를 담당합니다. 향후 하네스 서버를 위한 문서 원천 자료일 뿐입니다. 이 저장소에는 하네스 서버/런타임 구현, 런타임 홈, 생성된 상태 보기 시스템, 적합성 실행기, 런타임 상태, 런타임 데이터가 없습니다.

문서 파일은 런타임 상태가 아닙니다. 이 저장소의 Markdown 파일은 향후 하네스 규칙을 설명할 수는 있지만, 하네스 기록, 생성된 아티팩트, 상태 보기, 수락 기록, 증거 기록, 운영 파일이 아닙니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 제품 저장소, 하네스 서버 또는 설치 위치, 런타임 홈의 분리
- 제품 저장소 파일, 생성된 표시, 대화 텍스트, 커넥터 설명, 에이전트 기억이 하네스 권한을 만들지 않는다는 규칙
- 서버 설치 위치와 런타임 데이터 위치를 구분하는 규칙
- 런타임 홈이 자동으로 안전하거나 격리되거나 변조 방지된 공간은 아니라는 경계

이 문서는 담당하지 않습니다.

- 저장소 기록, 효과, 아티팩트, 버전 관리, 잠금, 마이그레이션: [참조 색인](README.md)의 저장소 담당 문서
- 공개 API 스키마나 메서드 동작: [참조 색인](README.md)의 API 담당 문서
- 보안 보장 의미나 자세한 보안 비주장: [보안](security.md)
- 상태 보기 권한: [상태 보기 권한 참조](projection-and-templates.md)

## 세 가지 위치

하네스 문서에서는 아래 세 위치를 분명히 나눠야 합니다.

| 위치 | 쉬운 의미 | 경계 규칙 |
|---|---|---|
| 제품 저장소 | 사용자의 프로젝트 작업 공간. | 제품 파일을 입력으로 제공할 수 있지만, 기본적으로 하네스 런타임 상태나 런타임 홈이 아닙니다. |
| 하네스 서버 또는 설치 위치 | 향후 서버 프로세스, 패키지, 설치된 애플리케이션 위치. | 하네스 API와 기록을 중재할 수 있지만, 설치 위치가 곧 런타임 데이터 위치는 아닙니다. |
| 런타임 홈 | 향후 하네스 기록, 로컬 저장소 메타데이터, 아티팩트 저장소를 담는 운영 데이터 공간. | 런타임 데이터 공간이지 제품 저장소가 아니며, OS 수준 보안의 증거도 아닙니다. |

## 제품 저장소

제품 저장소는 사용자의 프로젝트 작업 공간입니다. 제품 파일은 향후 하네스 확인이나 사용자 소유 판단의 입력이 될 수 있습니다. 하지만 제품 저장소 내용은 하네스 상태도, 생성된 하네스 출력도, 하네스 권한 증거도 아닙니다.

제품 저장소는 자동으로 런타임 홈이 되지 않습니다. 향후 구현이 프로젝트 안에 하네스 메타데이터를 둘 수 있게 하더라도, 그 메타데이터의 의미와 검증 방식, 그리고 일반 제품 파일이 하네스 기록으로 바뀌지 않는 이유는 저장소/런타임 담당 문서가 정의해야 합니다.

## 하네스 서버 또는 설치 위치

향후 하네스 서버는 하네스 기록과 API 동작을 중재합니다. 서버 설치 위치는 향후 구현에서 서버 코드, 패키지, 설정, 애플리케이션 리소스가 놓일 수 있는 곳입니다.

설치 위치와 런타임 데이터 위치를 섞어 말하면 안 됩니다. 어떤 디렉터리에서 서버를 설치하거나 실행한다고 해서 그 디렉터리가 런타임 홈이 되는 것은 아닙니다. 그렇게 보려면 저장소/런타임 담당 문서가 명시해야 합니다.

이 저장소에는 하네스 서버 구현이 없습니다. 문서 편집은 서버 코드를 만들지 않고, 런타임 동작을 시작하지 않으며, 제품/런타임 쓰기를 승인하지 않습니다.

## 런타임 홈

런타임 홈은 향후 사용자별 또는 설치별 운영 데이터 공간입니다. 하네스 소유 기록, 로컬 저장소 메타데이터, 스테이징되었거나 지속된 아티팩트 데이터, 잠금, 마이그레이션, 그 밖의 운영 데이터는 저장소/런타임 담당 문서가 정의한 경우에만 여기에 둘 수 있습니다.

런타임 홈은 자동으로 안전하거나 격리되거나 변조 방지된 공간이 아닙니다. 파일 권한, 해시, 진단, 소유자 확인은 담당 문서가 정의하면 방어적 확인으로 쓸 수 있습니다. 하지만 그것만으로 OS 수준 샌드박스, 임의 도구 제어, 실행 전 차단이 생기지는 않습니다.

이 문서 저장소는 런타임 홈이 아닙니다.

## 위치별 저장 가능 정보

| 위치 | 저장할 수 있는 것 | 그렇게 취급하면 안 되는 것 |
|---|---|---|
| 제품 저장소 | 제품 소스, 제품 문서, 테스트, 프로젝트 설정, 향후 하네스 확인이 살펴볼 수 있는 제품 파일. | 하네스 런타임 상태, 생성된 하네스 기록, 런타임 홈, 권한 증거. |
| 하네스 서버 또는 설치 위치 | 향후 서버 실행 코드, 설치된 패키지, 서버 설정, 애플리케이션 리소스. | 제품 작업 공간 내용, 기준 런타임 데이터, 런타임 홈이 존재한다는 증거. |
| 런타임 홈 | 저장소/런타임 담당 문서가 정의한 향후 하네스 운영 기록, 런타임 메타데이터, 로컬 저장소 데이터, 아티팩트, 잠금, 마이그레이션, 관련 진단. | 제품 소스, 서버 설치 파일, 변조 방지 저장소, OS 권한 경계. |
| 이 문서 저장소 | 향후 하네스 동작을 위한 원천 문서. | 런타임 상태, 서버 구현, 생성된 상태 보기, 증거, QA, 수락, 닫기 기록, 적합성 결과. |

## 추론하면 안 되는 것

- 이 저장소에 동작하는 하네스 서버, 런타임 홈, 런타임 데이터가 있다고 추론하면 안 됩니다.
- 문서 파일이 하네스 기록이나 생성된 운영 파일이라고 추론하면 안 됩니다.
- 담당 문서가 정한 런타임 설정이 없는 한 제품 저장소를 런타임 홈이라고 추론하면 안 됩니다.
- 서버 설치 디렉터리를 런타임 데이터 디렉터리라고 추론하면 안 됩니다.
- 런타임 홈이 변조 방지, 격리, 임의 로컬 도구로부터의 보호를 제공한다고 추론하면 안 됩니다.
- 제품 저장소 텍스트, 생성된 Markdown, 렌더링된 표시, 대화 텍스트, 커넥터 설명, 에이전트 기억, 복사된 `surface_id`, 표시된 `ArtifactRef`, 렌더링된 상태 보기에서 하네스 권한을 추론하면 안 됩니다.

## 보안 경계 링크

이 문서는 위치 경계와 금지되는 추론을 설명합니다. 보안 비주장은 경계를 정직하게 유지하는 데 필요한 만큼만 요약합니다. 현재 문서는 OS 수준 권한 제어, 임의 도구 샌드박스, 변조 방지 저장소, 기본 도구 실행 전 차단, 보안 격리, 권한 격리를 주장하지 않습니다.

자세한 보장 수준, 역량 조건부 탐지형 표현, 명시적 비주장, 이후 예방형 통제 요건은 [보안](security.md)이 담당합니다.
