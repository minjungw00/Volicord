# 저장소

이 문서는 저장소 문서 묶음의 짧은 경로 문서입니다. 참조 문서일 뿐이며 이 저장소에 Harness Server, Runtime Home, 데이터베이스, 아티팩트 저장소, 마이그레이션 실행기, 생성된 상태 보기, 런타임 상태, 런타임 DDL을 만들지 않습니다.

## 읽는 법

이 저장소 문서 묶음은 조건, 결과, 예외, 비주장을 분리해 설명합니다. 저장 효과를 API 스키마 형태나 사용자 표시 문구와 혼동하지 마세요.

- 조건: 저장 규칙이 적용되는 메서드 분기, 기록 상태, 아티팩트 생명주기 단계입니다.
- 결과: 저장되는 행, 아티팩트, 재실행 행, 이벤트, 상태 버전 영향입니다.
- 예외: 거절, `dry_run`, 읽기 전용, 호환성 실패처럼 저장 효과가 없거나 제한되는 분기입니다.
- 비주장: 응답 형태, 상태 보기, 보고서 문장, 경로 문자열이 저장 권한을 만들지 않는다는 경계입니다.

## 문서 묶음이 담당하는 것

저장소 문서 묶음은 하네스 기록이 어디에 지속되는지, 커밋된 기록이 저장소 권한으로서 무엇을 뜻하는지, 메서드 분기가 저장에 어떤 영향을 주는지, 아티팩트가 임시 스테이징에서 지속 참조로 어떻게 이동하는지, 프로젝트 전체 버전 관리, 멱등성, 잠금, 마이그레이션이 저장 계층에서 어떻게 동작하는지를 담당합니다.

저장소는 Core가 커밋하고 담당 Core/API/아티팩트/저장소 계약에 맞게 검증한 행이나 아티팩트 기록에 대해서만 권한이 됩니다. 대화, 생성된 Markdown, 상태 카드, 상태 보기, 커넥터 출력, 운영자 출력, 보고서 문장은 저장소 권한이 아닙니다.

## 담당하지 않는 것

저장소 문서는 API 요청 또는 응답 형태, 공개 오류 우선순위, 메서드 동작, Core 생명주기 의미, 보안 보장, Runtime Home 배포, 권한 주장을 담당하지 않습니다. 해당 계약은 [API 메서드](api/methods.md), 메서드 담당 문서, API 스키마 담당 문서, [Core 모델](core-model.md), [API 오류](api/errors.md), [보안](security.md), [런타임 경계](runtime-boundaries.md)를 봅니다.

API 데이터 형태와 저장 효과는 별개입니다.

- 조건: `CloseReadinessBlocker[]`, `ArtifactRef`, `StagedArtifactHandle` 같은 값이 API 응답에 포함될 수 있습니다.
- 결과: 이 값들은 API 데이터를 설명합니다.
- 비주장: 응답 필드가 있다는 사실만으로 행이 쓰였거나, 아티팩트가 승격되었거나, 스테이징 핸들이 소비되었거나, `project_state.state_version`이 바뀌었다고 볼 수 없습니다.
- 관련 담당 문서: 실제 저장 효과는 [저장 효과](storage-effects.md), 아티팩트 생명주기는 [아티팩트 저장소](storage-artifacts.md), 상태 버전 영향은 [저장소 버전 관리](storage-versioning.md)를 봅니다.

## 저장소 담당 경로

이 요약 표는 첫 경로 선택에만 사용합니다. 세부 블록은 조건을 긴 표 셀 밖에 둡니다.

| 필요 | 담당 문서 |
|---|---|
| 기록 배치 | [저장소 기록](storage-records.md) |
| 저장 효과 | [저장 효과](storage-effects.md) |
| 아티팩트 생명주기 | [아티팩트 저장소](storage-artifacts.md) |
| 버전 관리와 운영 경계 | [저장소 버전 관리](storage-versioning.md) |

**저장소 기록 경로**

조건:

- Runtime Home 배치를 확인해야 합니다.
- 로컬 저장소 가정을 확인해야 합니다.
- 지속 기록 범주와 테이블 개요를 확인해야 합니다.
- 저장소 소유 기록 값과 상태 필드를 확인해야 합니다.
- 저장소 소유 JSON과 기록 수준 현재/이후 제외 경계를 확인해야 합니다.

담당 문서:
- [저장소 기록](storage-records.md).

**저장 효과 경로**

조건:

- 읽기 전용 효과, `dry_run` 효과, 거절된 효과 없음 분기를 확인해야 합니다.
- 커밋된 차단 효과나 메서드별 지속 저장 효과를 확인해야 합니다.

담당 문서:
- [저장 효과](storage-effects.md).

**아티팩트 저장소 경로**

조건:

- 아티팩트 스테이징, `ArtifactRef`, 기존 아티팩트 연결을 확인해야 합니다.
- 스테이징 핸들 승격, 아티팩트 본문 읽기 경계, 보존, 무결성을 확인해야 합니다.

담당 문서:
- [아티팩트 저장소](storage-artifacts.md).

**저장소 버전 관리 경로**

조건:

- `project_state.state_version`을 확인해야 합니다.
- 멱등성, 재실행 행, 이벤트 의미, 잠금, 마이그레이션을 확인해야 합니다.

담당 문서:
- [저장소 버전 관리](storage-versioning.md).

저장소 담당 문서는 Harness Runtime Home 기록을 설명합니다. 이 문서 저장소는 Runtime Home이 아니며 생성된 런타임 상태를 담으면 안 됩니다.
