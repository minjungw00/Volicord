# 상태 보기 권한 참조

이 문서는 `Projection`(읽기 전용 상태 보기) 권한과 읽기 전용 파생 표시 경계를 담당합니다. 정확한 렌더링 본문 기대치는 [템플릿 본문](template-bodies.md)이 담당합니다.

참조 문서일 뿐입니다. 런타임 상태 보기, 런타임 상태, 생성 산출물, 증거 기록, QA 기록, 최종 수락 기록, 잔여 위험 기록, 닫기 기록, 구현 경로가 끝난 서버 계획이 아닙니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 상태 보기 권한 경계
- 읽기 전용 파생 표시 규칙
- 상태 보기 출력의 최신성 및 사용할 수 없음 표현
- 렌더링된 라벨은 기준 스키마 값이 아니라 표시 문구라는 규칙
- 상태 보기 조정, 지속 저장되는 상태 보기 작업, 관리 블록 불일치 복구의 이후 전용 경계

이 문서는 담당하지 않습니다.

- 상태 카드, 판단 요청, 실행/증거 요약, 닫기 결과, 에이전트 맥락 패킷 본문: [템플릿 본문](template-bodies.md)
- 원천 Core 상태: [Core 모델](core-model.md)
- 저장소 기록이나 상태 보기 저장소 후보: 저장소 담당 문서와 [범위 참조](scope.md)
- 공개 API 스키마: API 스키마 담당 문서
- 커넥터 동작: [에이전트 통합](agent-integration.md)

## 권한 경계

아래 항목이 기준입니다.

- Core가 소유한 상태
- 사용자 판단 기록
- 닫기 기록
- 잔여 위험 기록
- 저장소 행
- 지속 `ArtifactRef` 기록

상태 보기는 파생 표시일 뿐입니다. 담당 상태가 아닙니다.

아래 항목은 표시 또는 지원 맥락일 뿐입니다.

- 상태 보기
- 상태 카드
- Markdown 보고서
- 렌더링된 템플릿
- 대화 메시지
- 커넥터 출력
- 에이전트 맥락 패킷

렌더링된 출력은 담당 기록의 값을 인용하거나 요약하거나 링크할 수 있습니다. 하지만 두 번째 상태 저장소가 아니며, 잘 쓰였거나 사람이 고쳤거나 제품 저장소에 복사되었거나 에이전트 맥락에 주입되었다는 이유로 권한이 되지 않습니다.

## 렌더링된 표시가 할 수 없는 것

렌더링된 라벨, 상태 배지, Markdown 섹션, 상태 보기, 템플릿 본문, 대화 요약, 커넥터 출력, 에이전트 맥락 패킷은 그 자체로 아래 일을 할 수 없습니다.

- 쓰기 승인
- 증거 또는 지속 `ArtifactRef` 생성
- 검증, QA, 증거, 수락 같은 관문 충족
- 최종 수락 생성 또는 잔여 위험 수락
- 닫기 준비 상태 생성 또는 `CloseReadinessBlocker` 제거
- Task 닫기
- Core, 저장소, 아티팩트, 사용자 판단, 수락, 잔여 위험, 닫기 기록 변경

그런 결과에 대한 담당 기록이 실제로 있으면 표시는 그 기록을 보여 주거나 링크할 수 있습니다. 표시 문구가 그 결과의 근거가 되지는 않습니다.

## 파생 표시와 원천 상태

상태 보기 출력은 향후 담당 문서가 지속 저장되는 상태 보기 작업을 승격하지 않는 한 읽는 시점에 현재 담당 기록에서 계산합니다. 범위, 증거 공백, 차단 사유, 최신성, 다음 안전한 행동, 잔여 위험, 현재 보장 표현을 사람이 읽게 돕습니다.

생성된 표시는 생략, 가림 처리, 차단된 아티팩트, 사용할 수 없음 메모를 보존해야 합니다. 숨긴 원천 값을 재구성하면 안 됩니다. 필요한 담당 상태를 읽을 수 없으면 보기 좋은 상태를 꾸며 내지 말고 그 조건을 표시해야 합니다.

## 최신성과 원천 상태 경계

상태 보기 출력은 독자가 판단할 수 있을 만큼 원천 경계를 보여 줘야 합니다.

- 원천이 제공하면 상태 버전, 출처 참조, 관찰 시각 또는 그에 준하는 최신성 단서를 보여 줍니다.
- 오래됨, 일부만 있음, 사용할 수 없음, 충돌, 역량 제한 상태를 보존합니다.
- 표시 라벨을 기준 enum 값이나 스키마 필드와 구분합니다.
- 독자가 권한 기록을 확인해야 하면 관련 담당 문서로 연결합니다.
- 사람이 고쳤거나 오래된 표시는 버리거나 다시 계산할 표시로 다루며, Core 복구 입력으로 쓰지 않습니다.

## 템플릿 본문 담당 문서

[템플릿 본문](template-bodies.md)이 아래 본문 기대치를 담당합니다.

- 상태 카드
- 판단 요청
- 실행/증거 요약
- 닫기 결과
- 에이전트 맥락 패킷

이 문서는 그 담당 문서로 연결할 수 있지만, 해당 본문 섹션을 다시 정의하면 안 됩니다.

## 이후 경계

현재 MVP에는 활성 조정 큐, 편집 가능한 상태 보기 입력 경로, 상태 보기에서 Core로 되돌리는 복구 경로, 지속 저장되는 상태 보기 작업, 관리 블록 불일치 복구가 없습니다. 이런 항목은 범위, 대체 동작, 대체 불가능 규칙, 증명 기대와 함께 승격되기 전까지 이후 후보입니다.
