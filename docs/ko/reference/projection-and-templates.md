# 상태 보기와 템플릿 표시 경계

이 문서는 Volicord의 권한 기록과 읽기 전용 상태 보기, 상태 카드, 템플릿 보기의 차이를 담당합니다. `Projection`(읽기 전용 상태 보기)은 보기 또는 렌더링된 상태 표시이며 권한을 만들지 않습니다. 현재 렌더링 본문 지침, 표시 문구, 사용자 표시 라벨은 [템플릿 본문](template-bodies.md)이 담당합니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 상태 보기, 상태 카드, 템플릿 보기의 권한 경계
- 읽기 전용 파생 표시 규칙
- 표시 출력에서 원천 상태가 보여야 하는 기준
- 렌더링된 라벨은 기준 스키마 값이 아니라 표시 문구라는 규칙
- 권한 질문과 표시 문구 질문 사이의 경로 구분

이 문서는 담당하지 않습니다.

- 상태 카드, 판단 요청, 실행/증거 요약, 닫기 결과, 에이전트 맥락 패킷, 공개 오류 표시 문구: [템플릿 본문](template-bodies.md)
- 원천 Core 상태, 사용자 소유 판단, 증거, 수락 판단, 잔여 위험 판단, 닫기 준비 상태: [Core 모델](core-model.md)
- 저장소 기록, 아티팩트 기록, 저장 효과: [참조 색인](README.md)에서 고르는 저장소 담당 문서
- 공개 API 스키마나 메서드 동작: [참조 색인](README.md)에서 고르는 API 담당 문서
- 접점 등록, 현재 적용 접점 맥락, 역량 선언: [에이전트 통합](agent-integration.md)

## 권한 경계

권한은 렌더링된 보기가 아니라 담당 기록에 남습니다. 권한 기록에는 Core가 소유한 상태, 사용자 소유 판단, 담당 계약에 따라 기록된 증거와 아티팩트, 수락 판단, 잔여 위험 판단, 닫기 준비 상태, 저장소 문서가 담당하는 저장소 기록이 포함됩니다.

상태 보기, 상태 카드, 템플릿 보기는 읽기 전용 표시입니다. 담당 기록의 값을 인용하거나 요약하거나 연결할 수는 있습니다. 하지만 잘 쓰였거나, 사람이 고쳤거나, `Product Repository`에 복사되었거나, 에이전트 맥락에 주입되었다고 해서 두 번째 상태 저장소가 되지는 않습니다.

## 보기가 권한을 만들 수 없는 것

렌더링된 라벨, 상태 배지, Markdown 섹션, 상태 보기, 템플릿 본문, 대화 요약, 접점 출력, 에이전트 맥락 패킷은 그 자체로 아래 일을 할 수 없습니다.

- `Write Authorization` 생성
- 증거 또는 지속 `ArtifactRef` 생성
- 검증, QA, 증거, 수락 같은 관문 충족
- 최종 수락 생성 또는 잔여 위험 수락
- 닫기 준비 상태 생성 또는 `CloseReadinessBlocker` 제거
- `Task` 닫기
- Core, 저장소, 아티팩트, 사용자 판단, 수락 판단, 잔여 위험 판단, 닫기 준비 상태 변경

그런 결과에 대한 담당 기록이 실제로 있으면 보기는 그 기록을 보여 주거나 연결할 수 있습니다. 표시 문구가 그 결과의 근거가 되지는 않습니다.

## 표시 안의 원천 상태

파생 표시는 독자나 에이전트가 그 표시의 근거를 알 수 있을 만큼 원천 경계를 보여 줘야 합니다.

표시 출력은 아래 기준을 지킵니다.

- 담당 결과가 제공하면 출처 참조, `state_version`, 관찰 시각 또는 그에 준하는 원천 단서를 보여 줍니다.
- 오래됨, 일부만 있음, 사용할 수 없음, 가림 처리, 차단된 아티팩트, 충돌, 역량 제한 같은 원천 조건을 보존합니다.
- 표시 라벨을 기준 enum 값이나 스키마 필드와 구분합니다.
- 독자가 권한 기록을 확인해야 하면 관련 담당 문서로 연결합니다.
- 사람이 고쳤거나 오래된 표시는 버리거나 다시 계산할 표시로 다루며, Core 복구 입력으로 쓰지 않습니다.

## 템플릿과 라벨 경계

[템플릿 본문](template-bodies.md)은 상태 카드, 판단 요청, 실행/증거 요약, 닫기 결과, 에이전트 맥락 패킷, 공개 오류 표시 라벨의 현재 렌더링 본문 지침을 담당합니다.

이 문서는 어떤 보기가 권한인지 표시인지 말할 수 있습니다. 하지만 그 보기의 정확한 문구, 본문 섹션, 지역화 라벨을 정의하면 안 됩니다.

렌더링된 라벨은 독자가 담당 기록을 이해하도록 의미 담당 문서를 가리킬 수 있습니다. 하지만 그 의미를 다시 정의하거나 API 값, 저장소 필드, `ErrorCode` 값, 차단 사유 코드의 이름을 바꾸지는 않습니다.

## 담당 문서 링크

- [템플릿 본문](template-bodies.md)은 현재 렌더링 본문 지침, 표시 문구, 사용자 표시 라벨을 담당합니다.
- [Core 모델](core-model.md)은 Core 권한, 사용자 소유 판단, 닫기 준비 상태, 최종 수락, 잔여 위험 경계를 담당합니다.
- [참조 색인](README.md)은 API, 저장소, 아티팩트, 보안 담당 문서 질문을 안내합니다.
- [에이전트 통합](agent-integration.md)은 접점 등록, 현재 적용 접점 맥락, 역량 선언을 담당합니다.
