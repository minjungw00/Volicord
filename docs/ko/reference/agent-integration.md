# 에이전트 통합 참조

이 문서는 현재 문서 세트의 에이전트 커넥터 동작과 역량 맥락 경계를 담당합니다. 접점별 사용 레시피는 이 문서가 아니라 [접점별 사용 레시피](../use/surface-recipes.md)가 담당합니다.

이 문서는 문서 원천 자료일 뿐입니다. 커넥터, MCP 서버, 런타임 상태, 생성된 매니페스트, 적합성 실행기를 구현하지 않습니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 커넥터 경계에서의 `capability_profile` 의미
- 커넥터 경계에서의 `VerifiedSurfaceContext` 의미
- 확인된 역량 맥락에 따른 보장 표시 조건
- 맥락 주입/가져오기 지침
- 커넥터 대체 동작
- 커넥터 적합성 경계
- 에이전트 맥락에서 하나의 `doc_id`에 한 언어만 싣는 검색 지침

이 문서는 담당하지 않습니다.

- CLI, IDE/편집기, 채팅, 로컬 MCP 접점 레시피: [접점별 사용 레시피](../use/surface-recipes.md)
- API 메서드 동작, 요청 래퍼, 스키마 구조: [참조 색인](README.md)에서 고르는 API 담당 문서
- 저장소 배치, 아티팩트 생명주기, 스테이징 핸들 검증: [참조 색인](README.md)에서 고르는 저장소와 아티팩트 담당 문서
- 보안 보장 의미: [보안](security.md)
- Product Repository, Harness Server, Runtime Home 분리: [런타임 경계](runtime-boundaries.md)
- 정확한 템플릿 본문: [템플릿 본문](template-bodies.md)

## 커넥터 경계

커넥터는 하네스와 에이전트 접점 사이의 맥락을 전달합니다.

결과:

- 담당 경로가 정의한 하네스 상태를 요청할 수 있습니다.
- 담당 결과를 표시할 수 있습니다.
- 압축된 맥락을 에이전트에 전달할 수 있습니다.

비주장:

- 커넥터 설명, 생성 파일, 대화 텍스트, Product Repository 파일, 상태 보기, 에이전트 기억은 그 자체로 권한을 증명하지 않습니다.
- 산문이나 캐시된 표시 텍스트는 Core 상태, 사용자 소유 판단, `Write Authorization`, 증거 충분성, 아티팩트 권한, 닫기 준비 상태, 잔여 위험 수락, 보안 보장을 만들지 않습니다.

로컬 접점 권한은 API와 보안 담당 문서가 정의한 등록되고 확인된 접점 맥락에 달려 있습니다.

## `capability_profile`

`capability_profile`은 등록된 접점이 무엇을 지원할 수 있는지 설명하는 커넥터 담당 정보입니다.

담당 문서가 해당 개념을 활성으로 둘 때 `capability_profile`은 아래 항목을 설명할 수 있습니다.

- 접근 등급.
- 로컬 도달 가능성.
- 변경 경로 탐지.
- 아티팩트 스테이징 또는 본문 읽기 지원.
- 표시 역량.

조건:

- 보호된 읽기, 상태 변경, 아티팩트 동작, 탐지형 표시, 보장 수준 주장이 `capability_profile`에 기대려면 등록된 로컬 접점과 현재 요청에 맞는지 먼저 비교해야 합니다.
- 프로필 조건부 동작은 현재 범위 담당 문서와 해당 담당 문서가 범위, 대체 동작, 증명 기대치를 갖춰 승격하기 전까지 비활성입니다.

비주장:

- `capability_profile` 자체는 권한이 아닙니다.
- 오래되었거나 복사되었거나 생성되었거나 사용자가 말로 제공한 역량 설명은 이후 후보를 활성 기능으로 만들 수 없습니다.
- 같은 설명은 더 강한 보장 수준을 정당화할 수도 없습니다.

커넥터는 빠진 지원을 조용히 강한 주장으로 바꾸지 말고 사용 불가 또는 역량 제한으로 보여줘야 합니다.

## `VerifiedSurfaceContext`

`VerifiedSurfaceContext`는 요청이 선택한 `surface_id`를 등록된 로컬 접점 사실, 전송/세션/바인딩 증거, 접근 등급, 역량 상태와 맞춰 서버가 도출한 결과입니다.

조건:

- 커넥터는 선택자로 `surface_id`를 전달할 수 있습니다.
- 커넥터는 `verified=true`를 스스로 주장할 수 없습니다.
- 보호된 읽기와 상태 변경은 API 담당 문서가 해당 메서드와 확인된 맥락이 호환된다고 할 때만 접점에 의존할 수 있습니다.

결과:

- 공개 API 요청 하나에는 요청 수준 `VerifiedSurfaceContext.access_class`가 정확히 하나 있습니다.
- 아티팩트 입력 같은 중첩 페이로드는 두 번째 요청 접근 등급을 추가하지 않습니다.
- 스테이징된 아티팩트의 `created_by_surface_id`와 `created_by_surface_instance_id` 같은 서버 기록 출처는 `VerifiedSurfaceContext`에서 옵니다.

비주장:

- 호출자 산문은 서버 기록 출처가 아닙니다.
- 복사된 식별자, 생성된 Markdown, 대화 텍스트, 상태 보기 텍스트, 에이전트 기억은 확인된 맥락을 대신하지 않습니다.

정확한 요청 래퍼와 접근 등급 값은 [MVP API](api/mvp-api.md)와 [API 값 집합](api/schema-value-sets.md)이 담당합니다.

## 보장 표시 조건

보장 표시는 현재 문서화된 수준에서 시작합니다. 기본은 협력형입니다.

다음 조건을 모두 만족하면 제한적인 `detective` 표시를 할 수 있습니다.

- 관련 역량 확인이 통과했습니다.
- 보안 담당 문서가 그 표현을 허용합니다.
- 표시 범위가 이름 붙은 접점, 역량, 관찰 범위로 제한됩니다.

커넥터가 표시해야 하는 제한 조건은 아래와 같습니다.

- Core, MCP, 로컬 접근, 변경 경로 탐지, 아티팩트 접근, 그 밖의 필요한 역량을 사용할 수 없습니다.
- 필요한 역량이 오래되었거나 서로 맞지 않거나 충분하지 않습니다.

비주장:

- 접점 이름만으로 `detective`, `preventive`, `isolated`를 추론하지 않습니다.
- 상태 카드, 대화 요약, 렌더링된 상태 보기, 사용자 표현만으로 더 강한 보장 수준을 추론하지 않습니다.

보장 어휘와 비주장은 [보안](security.md)이 담당합니다. 현재 MVP 범위와 프로필 조건부 경계는 [현재 MVP 범위](active-mvp-scope.md)가 담당합니다.

## 맥락 주입과 가져오기

커넥터는 다음 행동에 쓸 만큼 최신인 경우 압축된 에이전트 맥락을 주입할 수 있습니다. 패킷에는 아래 항목만 둡니다.

- 현재 작업 요약.
- 활성 범위와 범위 밖.
- 관련 접점 상태.
- `state_version`.
- 대기 중인 사용자 소유 판단.
- 차단 사유.
- 다음 안전한 행동.
- 증거 공백.
- 아티팩트 가용성 요약.
- 닫기 준비 상태.
- 잔여 위험 상태.
- 보장 수준.
- 출처 참조와 최신성 정도.

커넥터는 다음 행동에 정확한 세부사항이 필요할 때만 담당 문서 섹션을 가져와야 합니다. 전체 스키마, DDL, 템플릿 본문, 과거 로그, 생성된 아티팩트, 전체 아티팩트 본문, 관련 없는 계약 자료, 이후 후보 목록, 같은 `doc_id`의 두 언어판은 주입하지 않습니다. 한영 문서 유지보수에서 의미 일치 검토가 필요한 경우만 예외입니다.

주입된 맥락 패킷이 오래되었거나 연결이 끊겼거나 현재 접점과 맞지 않으면, 커넥터는 에이전트가 그 맥락에 기대기 전에 담당 경로에서 새 결과를 요청하거나 오래됨 조건을 표시해야 합니다.

## 대체 동작

Core, MCP, 상태 보기 데이터, 로컬 접근, 아티팩트 접근, 역량을 사용할 수 없거나 충분하지 않으면 커넥터는 제한을 드러내야 합니다.

결과:

- 다음 안전한 행동을 관련 담당 문서로 보냅니다.
- 권한을 지어내지 않습니다.

담당 문서가 정의한 실패 의미를 사용합니다. 일반적인 경로는 아래와 같습니다.

- `MCP_UNAVAILABLE`: Core, MCP, 또는 필요한 접점 도달 가능성을 사용할 수 없습니다.
- `LOCAL_ACCESS_MISMATCH`: 도달 가능한 로컬 접근이 등록된 접점 기대와 맞지 않거나 철회되었습니다.
- `CAPABILITY_INSUFFICIENT`: 접점은 인식되었지만 필요한 접근 등급, 관찰, 아티팩트 역량, 보장 지원이 없습니다.

대체 동작은 정직하고 작아야 합니다.

- 다시 연결하거나 진단합니다.
- 역량 있는 접점으로 옮깁니다.
- 동작을 줄입니다.
- 상태를 새로고침합니다.
- 빠진 사용자 소유 판단을 요청합니다.

예외: 하네스 밖에서 계속하는 것은 사용자가 그 방식을 명시적으로 선택한 경우에만 가능합니다.

## 커넥터 적합성 경계

커넥터 적합성은 담당 문서가 정의한 결과를 보존하고 더 강하게 만들지 않는다는 뜻입니다. 적합한 커넥터는 아래를 지킵니다.

- 생성 텍스트나 대화 텍스트가 아니라 담당 경로에서 권한을 도출합니다.
- 요청 수준 `VerifiedSurfaceContext.access_class` 경계를 보존합니다.
- 사용할 수 없음, 불일치, 오래됨, 역량 부족 상태를 Core 기록을 만들어 내지 않고 보고합니다.
- 관련 담당 문서와 역량 확인이 뒷받침할 때만 보장 수준을 표시합니다.
- 사용자 소유 판단, 민감 동작 승인, 최종 수락, 잔여 위험 수락, 증거 충분성, 닫기 준비 상태를 서로 구분합니다.
- 접점 레시피는 이 참조 문서를 운영 매뉴얼로 바꾸지 않고 [접점별 사용 레시피](../use/surface-recipes.md)에 둡니다.

이 경계는 향후 커넥터 동작을 위한 문서 계약입니다. 실행 가능한 적합성 실행기가 아니며 생성된 적합성 결과를 만들지 않습니다.
