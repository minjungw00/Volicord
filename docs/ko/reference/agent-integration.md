# 에이전트 통합 참조

이 문서는 현재 문서 세트의 에이전트 커넥터 동작과 역량 맥락 경계를 담당합니다. 접점별 사용 레시피는 이 문서가 아니라 [접점별 사용 레시피](../use/surface-recipes.md)가 담당합니다.

이 문서는 참조 문서일 뿐입니다. 커넥터, MCP 서버, 런타임 상태, 생성된 매니페스트, 적합성 실행기를 구현하지 않습니다.

## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 커넥터 경계에서의 `capability_profile` 의미
- 커넥터 경계에서의 `VerifiedSurfaceContext` 의미
- 확인된 역량 맥락에 따른 보장 표시 조건
- 맥락 주입/가져오기 지침
- 커넥터 대체 동작
- 커넥터 적합성 경계
- 에이전트 맥락에서 하나의 `doc_id`에 한 언어만 싣는 검색 지침

이 문서는 담당하지 않습니다.

- CLI, IDE/편집기, 채팅, 로컬 MCP 접점 레시피: [접점별 사용 레시피](../use/surface-recipes.md)
- API 메서드 동작, 요청 래퍼, 스키마 구조: [참조 색인](README.md)에서 고르는 API 담당 문서
- 저장소 배치, 아티팩트 생명주기, 스테이징 핸들 검증: [참조 색인](README.md)에서 고르는 저장소와 아티팩트 담당 문서
- 보안 보장 의미: [보안](security.md)
- Product Repository, Harness Server, Runtime Home 분리: [런타임 경계](runtime-boundaries.md)
- 정확한 템플릿 본문: [템플릿 본문](template-bodies.md)

## 커넥터 경계

커넥터는 하네스와 에이전트 접점 사이의 맥락을 전달합니다.

조건:

- 커넥터는 담당 문서가 정의한 하네스 결과와 선택된 에이전트 접점 사이에서만 맥락을 전달합니다.
- 로컬 접점 권한은 API와 보안 담당 문서가 정의한 등록되고 확인된 접점 맥락에 달려 있습니다.

에이전트가 할 수 있는 것:

- 커넥터를 통해 담당 문서가 정의한 하네스 상태를 요청할 수 있습니다.
- 담당 결과를 표시할 수 있습니다.
- 압축된 맥락을 에이전트에 전달할 수 있습니다.

에이전트가 하면 안 되는 것:

- 커넥터 설명, 생성 파일, 대화 텍스트, Product Repository 파일, 상태 보기, 에이전트 기억을 그 자체로 권한으로 취급하면 안 됩니다.
- 산문이나 캐시된 표시 텍스트로 Core 상태, 사용자 소유 판단, `Write Authorization`, 증거 충분성, 아티팩트 권한, 닫기 준비 상태, 잔여 위험 수락, 보안 보장을 만들면 안 됩니다.

담당 문서 링크:

- [접점별 사용 레시피](../use/surface-recipes.md)는 접점별 레시피를 담당합니다.
- [런타임 경계](runtime-boundaries.md)는 Product Repository, Harness Server, Runtime Home 분리를 담당합니다.
- [보안](security.md)은 보안 보장 의미를 담당합니다.

## `capability_profile`

`capability_profile`은 등록된 접점이 무엇을 지원할 수 있는지 설명하는 커넥터 담당 정보입니다.

조건:

- 담당 문서가 어떤 개념을 활성으로 두어야 `capability_profile`이 그 개념을 지원 항목으로 설명할 수 있습니다.
- 보호된 읽기, 상태 변경, 아티팩트 동작, 탐지형 표시, 보장 수준 주장이 `capability_profile`에 기대려면 등록된 로컬 접점과 현재 요청에 맞는지 먼저 비교해야 합니다.
- 프로필 조건부 동작은 현재 범위 담당 문서와 해당 담당 문서가 범위, 대체 동작, 증명 기대치를 갖춰 승격하기 전까지 비활성입니다.

에이전트가 할 수 있는 것:

- 지원되는 접근 등급을 설명할 수 있습니다.
- 로컬 도달 가능성을 설명할 수 있습니다.
- 변경 경로 탐지를 설명할 수 있습니다.
- 아티팩트 스테이징 또는 본문 읽기 지원을 설명할 수 있습니다.
- 표시 역량을 설명할 수 있습니다.
- 빠진 지원을 사용 불가 또는 역량 제한으로 표시할 수 있습니다.

에이전트가 하면 안 되는 것:

- `capability_profile` 자체를 권한으로 취급하면 안 됩니다.
- 오래되었거나 복사되었거나 생성되었거나 사용자가 말로 제공한 역량 설명으로 지원 범위 밖 기능를 활성 기능으로 만들면 안 됩니다.
- 같은 설명으로 더 강한 보장 수준을 정당화하면 안 됩니다.

Fallback:

- 필요한 지원이 없거나 오래되었거나 맞지 않거나 충분하지 않으면 사용 불가 또는 역량 제한 상태로 표시하고 다음 판단을 관련 담당 문서 결과로 보냅니다.

담당 문서 링크:

- [기준 범위](scope.md)는 활성 범위와 프로필 조건부 범위 경계를 담당합니다.
- [보안](security.md)은 보장 어휘와 보장 강도 비주장을 담당합니다.
- [API 값 집합](api/schema-value-sets.md)은 접근 등급 값 이름을 담당합니다.

## 로컬 접점 등록

로컬 접점 등록은 서버가 확인된 접점 맥락을 도출할 때 쓰는 사실을 제공합니다. 커넥터는 선택자를 전달하고 담당 결과를 표시할 수 있지만 로컬 권한을 만들지는 않습니다.

조건:

- 요청은 등록된 로컬 접점을 `surface_id`로 선택할 수 있습니다.
- 서버가 선택된 접점을 등록된 로컬 사실, 전송/세션/바인딩 증거, 접근 등급, 역량 상태와 맞춥니다. 커넥터가 그 확인을 대신하지 않습니다.
- 보호된 읽기, 상태 변경, 아티팩트 동작은 메서드 담당 문서가 확인된 맥락과 호환된다고 할 때만 접점에 의존할 수 있습니다.

에이전트가 할 수 있는 것:

- `surface_id`를 선택자로 전달할 수 있습니다.
- 담당 문서가 반환한 사용 불가, 불일치, 오래됨, 역량 부족 접점 상태를 표시할 수 있습니다.

에이전트가 하면 안 되는 것:

- 호출자 산문, 복사된 식별자, 생성된 Markdown, 대화 텍스트, 상태 보기 텍스트, 에이전트 기억으로 로컬 도달 가능성, 접근 등급, `verified=true`, 스테이징된 아티팩트 출처를 주장하면 안 됩니다.
- `surface_id`, `surface_instance_id`, 접점 이름을 권한 증거로 취급하면 안 됩니다.

Fallback:

- 등록된 사실을 현재 요청과 맞출 수 없으면 담당 메서드가 호환되는 확인 맥락을 반환할 때까지 로컬 접점을 사용 불가, 불일치, 오래됨, 또는 역량 부족 상태로 표시합니다.

담당 문서 링크:

- [API 메서드](api/methods.md)와 메서드 담당 문서는 메서드 요청 조건을 담당합니다.
- [API 값 집합](api/schema-value-sets.md)은 접근 등급 값을 담당합니다.
- [보안](security.md)은 접근 경계와 보장 표현을 담당합니다.

## `VerifiedSurfaceContext`

`VerifiedSurfaceContext`는 요청이 선택한 `surface_id`를 등록된 로컬 접점 사실, 전송/세션/바인딩 증거, 접근 등급, 역량 상태와 맞춰 서버가 도출하는 결과입니다.

조건:

- 공개 API 요청 하나에는 요청 수준 `VerifiedSurfaceContext.access_class`가 정확히 하나 있습니다.
- 아티팩트 입력 같은 중첩 페이로드는 두 번째 요청 접근 등급을 추가하지 않습니다.
- 스테이징된 아티팩트의 `created_by_surface_id`와 `created_by_surface_instance_id` 같은 서버 기록 출처는 `VerifiedSurfaceContext`에서 옵니다.
- 보호된 읽기와 상태 변경은 API 담당 문서가 해당 메서드와 확인된 맥락이 호환된다고 할 때만 접점에 의존할 수 있습니다.

에이전트가 할 수 있는 것:

- 호환되는 요청 수준 `VerifiedSurfaceContext`를 포함한 담당 결과를 사용할 수 있습니다.
- 맥락을 표시하거나 전달할 때 요청 수준 `VerifiedSurfaceContext.access_class`를 보존할 수 있습니다.

에이전트가 하면 안 되는 것:

- `verified=true`를 스스로 주장하면 안 됩니다.
- 호출자 산문으로 스테이징된 아티팩트 출처를 제공하면 안 됩니다.
- 복사된 식별자, 생성된 Markdown, 대화 텍스트, 상태 보기 텍스트, 에이전트 기억을 확인된 맥락의 대체물로 쓰면 안 됩니다.

Fallback:

- 확인된 맥락이 없거나 메서드와 호환되지 않으면 그 접점에 기대지 말고 관련 사용 불가, 불일치, 오래됨, 또는 역량 부족 상태를 표시합니다.

담당 문서 링크:

- 정확한 요청 래퍼와 접근 등급 값은 [API 메서드](api/methods.md), 메서드 담당 문서, [API 값 집합](api/schema-value-sets.md)이 담당합니다.

## 보장 표시 조건

보장 표시는 현재 문서화된 수준에서 시작합니다. 기본은 협력형입니다.

조건:

- 제한적인 `detective` 표시에는 관련 역량 확인 통과가 필요합니다.
- 제한적인 `detective` 표시에는 보안 담당 문서가 그 표현을 허용해야 합니다.
- 표시 범위는 이름 붙은 접점, 역량, 관찰 범위로 제한되어야 합니다.

에이전트가 할 수 있는 것:

- 위 조건을 모두 만족할 때만 제한적인 `detective` 표현을 표시할 수 있습니다.
- Core, MCP, 로컬 접근, 변경 경로 탐지, 아티팩트 접근, 그 밖의 필요한 역량을 사용할 수 없다는 제한 조건을 표시할 수 있습니다.
- 필요한 역량이 오래되었거나 서로 맞지 않거나 충분하지 않다는 제한 조건을 표시할 수 있습니다.

에이전트가 하면 안 되는 것:

- 접점 이름만으로 `detective`, `preventive`, `isolated`를 추론하면 안 됩니다.
- 상태 카드, 대화 요약, 렌더링된 상태 보기, 사용자 표현만으로 더 강한 보장 수준을 추론하면 안 됩니다.

Fallback:

- 관련 역량이 표시된 보장을 뒷받침할 수 없으면 주장을 강하게 만들지 말고 사용 불가 또는 역량 제한으로 표시합니다.

담당 문서 링크:

- [보안](security.md)은 보장 어휘와 비주장을 담당합니다.
- [기준 범위](scope.md)는 기준 범위와 프로필 조건부 경계를 담당합니다.

## 맥락 주입과 가져오기

조건:

- 커넥터는 다음 행동에 쓸 만큼 최신이고 현재 접점과 호환되는 경우에만 압축된 에이전트 맥락을 주입할 수 있습니다.
- 커넥터는 다음 행동에 정확한 세부사항이 필요할 때만 담당 문서 섹션을 가져와야 합니다.

에이전트가 할 수 있는 것:

- 아래 항목만 담은 압축된 맥락 패킷을 주입할 수 있습니다.
  - 현재 작업 요약.
  - 활성 범위와 범위 밖.
  - 관련 접점 상태.
  - `state_version`.
  - 대기 중인 사용자 소유 판단.
  - 차단 사유.
  - 다음 안전한 행동.
  - 증거 공백.
  - 아티팩트 가용성 요약.
  - 닫기 준비 상태.
  - 잔여 위험 상태.
  - 보장 수준.
  - 출처 참조와 최신성 정도.
- 다음 행동에 필요한 정확한 담당 문서 섹션을 가져올 수 있습니다.

에이전트가 하면 안 되는 것:

- 전체 스키마를 주입하면 안 됩니다.
- DDL을 주입하면 안 됩니다.
- 템플릿 본문을 주입하면 안 됩니다.
- 과거 로그를 주입하면 안 됩니다.
- 생성된 아티팩트를 주입하면 안 됩니다.
- 전체 아티팩트 본문을 주입하면 안 됩니다.
- 관련 없는 계약 자료를 주입하면 안 됩니다.
- 지원 범위 밖 기능 목록을 주입하면 안 됩니다.
- 한영 문서 유지보수에서 의미 일치 검토가 필요한 경우가 아니라면 같은 `doc_id`의 두 언어판을 함께 주입하면 안 됩니다.

Fallback:

- 주입된 맥락 패킷이 오래되었거나 연결이 끊겼거나 현재 접점과 맞지 않으면 에이전트가 그 맥락에 기대기 전에 담당 경로에서 새 결과를 요청하거나 오래됨 조건을 표시합니다.

담당 문서 링크:

- [참조 색인](README.md)은 정확한 담당 문서 섹션 경로를 안내합니다.
- [접점별 사용 레시피](../use/surface-recipes.md)는 접점별 사용 레시피를 담당합니다.
- [번역 가이드](../maintain/translation-guide.md)는 한영 의미 일치 검토 기대치를 담당합니다.

## 대체 동작

조건:

- Core, MCP, 상태 보기 데이터, 로컬 접근, 변경 경로 탐지, 아티팩트 접근, 또는 필요한 역량을 사용할 수 없습니다.
- 필요한 역량이 오래되었거나 서로 맞지 않거나 충분하지 않습니다.

에이전트가 할 수 있는 것:

- 다시 연결하거나 진단할 수 있습니다.
- 역량 있는 접점으로 옮길 수 있습니다.
- 동작을 줄일 수 있습니다.
- 상태를 새로고침할 수 있습니다.
- 빠진 사용자 소유 판단을 요청할 수 있습니다.
- 사용자가 그 방식을 명시적으로 선택한 경우에만 하네스 밖에서 계속할 수 있습니다.

에이전트가 하면 안 되는 것:

- 권한을 지어내면 안 됩니다.
- 사용 불가, 불일치, 오래됨, 역량 부족 상태를 일반 성공 문구 속에 숨기면 안 됩니다.
- 사용자의 명시적 선택 없이 하네스 밖에서 계속하면 안 됩니다.

Fallback:

- 제한을 드러내고 다음 안전한 행동을 관련 담당 문서로 보냅니다.
- 담당 문서가 정의한 실패 의미를 사용합니다. 일반적인 경로는 아래와 같습니다.
  - `MCP_UNAVAILABLE`: Core, MCP, 또는 필요한 접점 도달 가능성을 사용할 수 없습니다.
  - `LOCAL_ACCESS_MISMATCH`: 도달 가능한 로컬 접근이 등록된 접점 기대와 맞지 않거나 철회되었습니다.
  - `CAPABILITY_INSUFFICIENT`: 접점은 인식되었지만 필요한 접근 등급, 관찰, 아티팩트 역량, 보장 지원이 없습니다.

담당 문서 링크:

- [API 오류](api/errors.md)는 공개 오류 코드 경로를 담당합니다.
- [접점별 사용 레시피](../use/surface-recipes.md)는 실무적인 접점 실패 요약을 담당합니다.
- [보안](security.md)은 보장 제한 표현을 담당합니다.

## 커넥터 적합성 경계

커넥터 적합성은 담당 문서가 정의한 결과를 보존하고 더 강하게 만들지 않는다는 뜻입니다. 적합한 커넥터는 아래를 지킵니다.

- 생성 텍스트나 대화 텍스트가 아니라 담당 경로에서 권한을 도출합니다.
- 요청 수준 `VerifiedSurfaceContext.access_class` 경계를 보존합니다.
- 사용할 수 없음, 불일치, 오래됨, 역량 부족 상태를 Core 기록을 만들어 내지 않고 보고합니다.
- 관련 담당 문서와 역량 확인이 뒷받침할 때만 보장 수준을 표시합니다.
- 사용자 소유 판단, 민감 동작 승인, 최종 수락, 잔여 위험 수락, 증거 충분성, 닫기 준비 상태를 서로 구분합니다.
- 접점 레시피는 이 참조 문서를 운영 매뉴얼로 바꾸지 않고 [접점별 사용 레시피](../use/surface-recipes.md)에 둡니다.

이 경계는 커넥터 동작을 위한 문서 계약입니다. 실행 가능한 적합성 실행기가 아니며 생성된 적합성 결과를 만들지 않습니다.
