# 범위 참조

이 참조 문서는 하네스의 지원되는 기준 범위 경계를 담당합니다. 기준 범위 안에 있는 기능 묶음, 기준 범위 밖에 남는 기능, 예약된 값과 프로필 조건부 값이 범위에 미치는 영향을 정의합니다.

이 페이지는 안정적인 참조 계약이며, 프로젝트 경과 서술이나 구현 계획이 아닙니다.

<a id="owns-does-not-own"></a>
## 담당하는 것 / 담당하지 않는 것

이 문서가 담당합니다.

- 지원되는 기준 범위 경계
- 포함되는 기능 묶음과 제외되는 기능 묶음
- 예약된 값과 프로필 조건부 값의 범위상 의미
- 다른 문서가 반복하지 말고 요약해 연결해야 하는 범위 수준 보장과 비주장 문구

이 문서가 담당하지 않습니다.

- 기준 범위 구현 읽기 경로. [구현 가이드](../build/implementation-guide.md)를 봅니다.
- API 메서드 동작
- 스키마 필드
- 저장소 기록이나 저장 효과
- 런타임과 제품 저장소 경계의 세부사항
- 보안 보장 의미나 증명 요구사항
- 적합성 절차
- 템플릿 본문
- 커넥터 동작
- 지원 범위 밖 기능의 상세 명세

어떤 기능이 지원되는 기준 범위에 속하는지 판단할 때는 이 문서를 기준으로 삼습니다. 경로 문서, 빌드 문서, README, 참조 문서는 상세 범위 목록을 반복하지 말고 이 문서로 연결합니다.

<a id="supported-baseline-scope"></a>
## 지원되는 기준 범위

기준 범위는 의도적으로 좁습니다. 이 문서가 기능 묶음을 포함하고 관련 담당 문서가 동작, 데이터 형태, 저장 또는 보안 결과를 정의할 때만 그 기능은 지원되는 기준 범위 안에 있습니다.

<a id="what-supported-means"></a>
### 지원된다는 뜻

이 문서 세트에서 지원된다는 것은 아래를 뜻합니다.

- 그 기능은 에이전트, 구현자, 유지보수자가 의존할 수 있는 기준 계약에 속합니다.
- 필요한 경우 동작, 스키마 형태, 저장 효과, 보안 결과, 용어를 정의하는 담당 문서가 있습니다.
- 다른 문서는 기능을 요약할 수 있지만, 세부 질문은 이 문서나 [참조 색인](README.md)에 나열된 담당 문서로 보내야 합니다.

지원된다는 것은 아래를 뜻하지 않습니다.

- 스키마나 예시에 등장하는 모든 값 이름이 기준 범위 동작으로 제공된다는 뜻이 아닙니다.
- 경로 문서, 예시, 스키마 메모, 값 집합이 지원 범위 밖 기능을 언급한다고 해서 그 기능이 지원되는 것은 아닙니다.
- 관련 담당 문서가 정의하지 않은 보안, 적합성, 저장소, 상태 보기 보장이 존재한다는 뜻이 아닙니다.

<a id="excluded-from-baseline-scope"></a>
## 기준 범위에 포함되지 않는 것

아래 기능 묶음은 기준 범위 밖에 있어 지원되지 않습니다. 이 범위 참조가 그 기능 묶음을 포함하고 영향받는 기존 담당 문서가 그 기능 묶음의 동작을 정의할 때만 지원됩니다.

제외된 기능:

- 접점 자체 아티팩트 캡처
- 지속 저장되는 상태 보기 작업, 상태 보기 조정, 생성된 상태 보기 파일, 관리되는 상태 보기 복구
- 확장 또는 추가 증거 수집 흐름
- 수동 QA와 외부 검증 작업 흐름
- 명령, 네트워크, 비밀값 접근 관찰
- 명령, 네트워크, 비밀값의 도구 실행 전 차단
- 예방형 격리, 샌드박싱, 또는 더 강한 격리 보장 의미
- 호스팅 대시보드
- 커넥터 마켓플레이스
- 내보내기 또는 전달 패키지
- 실행 가능한 픽스처 실행기
- 생성된 적합성 산출물
- 운영 프로필

범위 규칙:

- 여기에 나열된 기능은 기준 범위 밖에 있습니다.
- 제외된 기능은 기준 범위 요구사항이 아닙니다.
- 이 범위 참조와 영향받는 기존 담당 문서가 지원된다고 명시적으로 정의하지 않는 한 제외된 기능은 지원 동작이 아닙니다.
- 민감 동작 승인은 그 자체로 광범위한 보안 모니터링, 격리 보관, QA 게이트, 검증 게이트, 명령/네트워크/비밀값 관찰, 도구 실행 전 차단, 격리, 또는 그 밖의 지원 범위 밖 동작을 뜻하지 않습니다.

담당 문서 경로:

- API 메서드 동작, 스키마, 값 이름: [API 메서드](api/methods.md), [API 코어 스키마](api/schema-core.md), [API 상태 스키마](api/schema-state.md), [API 아티팩트 스키마](api/schema-artifacts.md), [API 판단 스키마](api/schema-judgment.md), [API 값 집합](api/schema-value-sets.md).
- 저장소 기록/효과와 아티팩트 생명주기: [저장소](storage.md), [저장소 기록](storage-records.md), [저장 효과](storage-effects.md), [아티팩트 저장소](storage-artifacts.md).
- 런타임과 제품 저장소 경계: [런타임 경계](runtime-boundaries.md).
- 보안 비주장, 보장 수준, 관찰, 차단, 격리, 민감 동작 승인 경계: [보안](security.md), [Core 모델](core-model.md).

<a id="reserved-and-profile-gated-values"></a>
## 예약된 값과 프로필 조건부 값

일부 값 이름은 예약된 값이나 프로필 조건부 값으로 존재할 수 있지만, 그것만으로 사용자에게 보이는 지원 기능이 되지는 않습니다.

예약된 값:

- 예약된 값은 어휘, 호환성 접점, 또는 값 집합 항목으로 나타날 수 있습니다.
- 예약된 값은 이름만으로 동작을 켜거나 기본값을 만들거나 닫기 준비 상태를 충족하거나 보장을 만들지 않습니다.

프로필 조건부 값:

- 프로필 조건부 값은 관련 프로필이나 게이트가 담당 문서에 정의되고 이 범위 참조가 그 결과 기능을 포함할 때만 사용할 수 있습니다.
- 프로필/게이트나 기능 담당 문서 중 하나라도 없으면 그 값은 지원되는 기준 범위 동작이 아닙니다.

사용 지점 규칙:

- 예약된 값과 프로필 조건부 값은 나타나는 자리에서 그렇게 표시합니다.
- 이 문서와 의미 담당 문서가 모두 그 동작을 정의하지 않는 한 기본값, 필수값, 지원됨, 강제됨, 수락됨, 검증됨, 닫기 준비 상태, 탐지형, 샌드박싱, 더 강한 격리 동작처럼 설명하지 않습니다.

담당 문서 링크:

- 정확한 값 이름과 값 집합 위치: [API 값 집합](api/schema-value-sets.md).
- 더 강한 격리 주장을 포함한 보장 의미와 비주장: [보안](security.md).
- 예약된 값과 프로필 조건부 값의 제품 용어: [용어집](glossary.md).

<a id="out-of-scope-capability-promotion"></a>
## 지원 범위 밖 기능의 승격

지원 범위 밖 기능은 이 범위 참조와 영향받는 담당 문서가 좁은 지원 계약을 정의할 때만 지원됩니다.

승격은 아래를 정의해야 합니다.

- 지원되는 동작과 대체 동작
- 책임지는 담당 문서
- 영향을 받는 경우 API 동작, 스키마, 저장 효과, 런타임 경계, 보안 보장, 적합성 점검, 용어 갱신
- 영어와 한국어 대응 문서

지원하려는 기능의 적용되는 담당 문서가 없다면 지원 기능으로 설명하기 전에 담당 문서를 만들거나 지정해야 합니다. 자리표시자를 기존 담당 문서처럼 독자에게 안내하지 않습니다.

승격 경계:

- 제외된 기능, 예약된 값, 프로필 조건부 기능이 예시, 경로 문구, 스키마 메모, 값 집합, 이 참조 문서에 언급된다는 사실만으로는 승격되지 않습니다.

<a id="supported-guarantee-boundary"></a>
## 지원되는 보장 경계

기준 범위는 [보안](security.md)이 정의하는 보장 수준을 사용합니다. 어떤 보장을 지원된다고 설명하려면 범위상 사용 가능성과 보안 의미가 서로 맞아야 합니다.

지원되는 경계:

- 이 문서와 [보안](security.md)이 다른 지원 보장을 정의하지 않는 한 기준 범위의 보장 경계는 `cooperative`입니다.
- `harness.prepare_write`와 `Write Authorization`은 제품 파일 쓰기 호환성 메커니즘이며 격리나 샌드박싱 보장이 아닙니다.

포함되지 않는 것:

- 기준 범위는 더 강한 격리 보장 의미를 제공하지 않습니다.
- 예약된 값이나 프로필 조건부 값인 보장 라벨은 기준 범위를 확장하지 않습니다.

담당 문서 링크:

- 보장 의미, 탐지형 표현, 보안 비주장: [보안](security.md).
- 보장 라벨 값 항목: [API 값 집합](api/schema-value-sets.md).
- 메서드 동작: [쓰기 준비 메서드](api/method-prepare-write.md), [API 메서드](api/methods.md)의 안내.
- Core 의미: [Core 모델](core-model.md).
