# Harness documentation

This is the English entry point for the Harness documentation set. It helps readers choose the right page; it does not define technical contracts.

## Reading paths

- New user: start with [Start](start.md), then continue to the [User Guide](use/user-guide.md).
- Working user: use the [User Guide](use/user-guide.md), [Judgment Examples](use/judgment-examples.md), and [Scope](reference/scope.md).
- Agent author/operator: read [AGENTS.md](../../AGENTS.md), then use [doc-index.yaml](../doc-index.yaml), the [Agent Guide](use/agent-guide.md), and [Agent Integration Reference](reference/agent-integration.md).
- Implementer: check [Scope](reference/scope.md) first, then use the [Reference Index](reference/README.md) to find exact contracts.
- Documentation maintainer: use the [Authoring Guide](maintain/authoring-guide.md), [Checks](maintain/checks.md), [Translation Guide](maintain/translation-guide.md), [doc-index.yaml](../doc-index.yaml), and [Terminology Map](../terminology-map.yaml).

## Current scope

[Scope](reference/scope.md) is the entry point for what is currently in or out of the MVP. Out-of-scope capabilities are not active requirements unless that owner says they have been promoted.

## Exact contracts

Use the [Reference Index](reference/README.md) when you need API, storage, security, schema, projection, or close-readiness owner documents. This README stays a route, not a shortcut copy of those contracts.
